## Блок 2: «Как работать с этим уроком» (текущее название: «Дополнительный блок»)

### Что пишет заказчик

**Назначение:**
> «Мета-блок для ученика, задающий рамку работы с уроком. Используется для:
> – управления ожиданиями ученика;
> – объяснения формата урока;
> – обозначения границ («о чём урок НЕ будет»);
> – снижения тревоги перед сложной/объёмной темой.
> Тип контента: инструктивный, не учебный.»

**Текущий статус:**
> «В текущей версии реализован как «Дополнительный блок»:
> – обычный блок урока;
> – отображается первым;
> – позволяет добавлять любые элементы (теория, практика, тесты и т.д.);
> – не имеет чёткого объяснения назначения;
> – по статусу равен основному блоку.»

**Результаты тестирования:**
> «Фаундер: активно использовался как методическая опора, повышает качество урока.
> Автор 1 (иностранный педагог): использует интуитивно, не до конца понимает статус блока, есть сомнение «что сюда писать».
> Автор 2 (педагог): блок полезен по смыслу, но создаёт путаницу из-за названия и равноправного положения среди учебных блоков.
> Автор 3 (юрист): не понимает назначение блока, игнорирует его заполнение.»

**Наблюдение:**
> «Автор не может качественно писать мета-блоки до тех пор, пока не закончил основной контент.»

**Предложения по доработке (цитата):**
> «1. Переименование блока – заменить название «Дополнительный блок» на «Как работать с этим уроком».
> 2. Изменение статуса блока – выделить блок в отдельный тип (мета-блок), не равный учебным блокам.
> 3. Ограничение функционала – ограничить типы элементов: разрешён только инструктивный контент; блок не предназначен для практики, тестов, примеров.
> 4. Разделение порядка заполнения и отображения – блок исключается из стартового сценария создания урока; в конструкторе не предлагается к заполнению первым.
> 5. Порядок отображения для ученика – в опубликованном уроке блок всегда отображается первым.
> 6. Визуальное отличие – визуально отделить блок от учебных (иной фон / рамка / оформление).
> 7. Активация блока и ИИ-поддержка – блок становится доступным после логического завершения основного учебного контента; при активации система предлагает автору создать мета-блок с опорой на один из методических шаблонов.»

**Требования к ИИ-поддержке:**
> «Шаблоны формулируются от лица автора к ученику и ориентированы на улучшение восприятия и усвоения материала. ИИ использует выбранный шаблон и содержание урока для формирования чернового текста. ИИ не изменяет и не перезаписывает авторский текст без явного действия автора.»

**Условие генерации:**
> «Генерация текста мета-блока становится доступной только после достижения минимального объёма осмысленного учебного контента (порог по количеству символов).»

```typescript
export function canActivateMetaBlock(blocks: CourseBlock[]): boolean {
  // Подсчитываем символы в основных учебных блоках
  const educationalBlocks = blocks.filter(b => b.category === 'educational')
  const totalChars = educationalBlocks.reduce((sum, block) => {
    return sum + block.elements.reduce((s, el) => s + (el.content?.length || 0), 0)
  }, 0)

  const MIN_CONTENT_THRESHOLD = 500  // минимум 500 символов
  return totalChars >= MIN_CONTENT_THRESHOLD
}
```

## Блок 3: Навигация

### Что пишет заказчик

**Назначение:**
> «Сервисный мета-элемент для ученика. Краткая карта урока: что будет происходить дальше и в какой логике. Используется для снижения неопределённости, ориентации в структуре урока и поддержки внимания в сложных и длинных уроках.»

**Текущий статус:**
> «Реализована как отдельный блок урока:
> – находится в общем списке блоков;
> – выглядит обязательной;
> – заполняется вручную;
> – предлагается автору на раннем этапе;
> – функционально равна учебным блокам.»

**Результаты тестирования:**
> «Фаундер: неочевидно назначение, ощущается дублирование структуры.
> Автор 1: использует интуитивно, сомневается, зачем нужен блок и что в нём писать.
> Автор 2: блок отвлекает, создаёт ощущение навязанного шаблона, перегружает интерфейс.
> Автор 3: назначение блока неочевидно, заполнение игнорируется.
> Общее: навигация полезна только в сложных/длинных уроках и плохо работает как обязательный ручной блок.»

**Наблюдение:**
> «При ручном и обязательном использовании навигация дублирует структуру урока, увеличивает когнитивную нагрузку автора и не даёт стабильной ценности ученику.»

**Предложения по доработке:**
> «1) Убрать навигацию из линейного списка блоков.
> 2) Перенести навигацию в финальный этап проектирования.
> 3) Закрепить статус сервисного мета-элемента.
> 4) Сделать навигацию опциональной.
> 5) Реализовать генерацию от структуры урока по запросу автора.
> 6) Зафиксировать формат: только текст, без интерактива.»

**Дополнительная информация:**
> «Короткий урок — если выполняются ВСЕ условия: 1 основной блок, не более 1 практики, не более 1 проверки знаний. В коротких уроках блок «Навигация» по умолчанию скрыт и не предлагается автору для генерации.»

```typescript
// lib/block-activation.ts
export function isShortLesson(blocks: CourseBlock[]): boolean {
  const educationalBlocks = blocks.filter(b => b.category === 'educational')
  const mainBlocks = educationalBlocks.filter(b => b.type.includes('main') || b.type === 'explanation')
  const practiceBlocks = educationalBlocks.filter(b => b.type === 'practice')
  const testBlocks = educationalBlocks.filter(b => b.type === 'knowledge_check')

  return mainBlocks.length <= 1 && practiceBlocks.length <= 1 && testBlocks.length <= 1
}

export function shouldShowNavigation(blocks: CourseBlock[]): boolean {
  return !isShortLesson(blocks)
}
```

## Блок 4: Тезисы (НОВЫЙ)

### Что пишет заказчик

**Назначение:**
> «Вспомогательное поле для первичной выгрузки смыслов автором. Используется для фиксации основных мыслей, идей и логики темы в свободной форме. Тип контента: служебный (не учебный). Адресат: автор.»

**Текущий статус:**
> «Отсутствует как отдельный элемент. В текущей версии автор сразу работает с учебным блоком.»

**Результаты тестирования:**
> «Авторы отмечают затык на старте, страх формы и "переписывание" вместо осмысленного изложения. Голосовая фиксация мыслей воспринимается как более естественная.»

**Наблюдение:**
> «Авторам проще сначала зафиксировать смысл и логику темы (тезисно или голосом), а уже затем разворачивать объяснение.»

**Требования к реализации:**
> «Отображается слева от учебного блока в виде отдельного поля. Всегда видно при работе с парным учебным блоком. Содержит поясняющую легенду о назначении поля.»

**Активация и логика заполнения:**
> «Доступно сразу при создании учебного блока. Заполнение свободное. Поддерживает текстовый ввод и загрузку/запись аудио с автоматической транскрипцией в текст.»

**ИИ-поддержка:**
> «ИИ по запросу автора может помочь осмыслить поток: предложить группировку тезисов, подсветить повторы. ИИ не переписывает и не изменяет тезисы. Все операции ИИ выполняются только по запросу автора и не изменяют исходный текст без подтверждения.»

**Отображение для ученика:**
> «Не отображается ученику ни в каком виде.»

**Предложения по доработке:**
> «1) Добавить отдельное поле «Тезисы».
> 2) Поддержать текст + аудио с транскрипцией.
> 3) Добавить поясняющую легенду.
> 4) Зафиксировать, что тезисы не являются учебным контентом.»

### Как реализовано сейчас

**Не реализовано.** Авторы сразу работают с основным блоком без предварительной выгрузки мыслей.

> «При отсутствии финального блока ученики хуже осознают, что именно усвоили, и воспринимают урок как незавершённый фрагмент.»

**Наблюдение:**
> «Эффективное завершение усиливает закрепление знаний и повышает ощущение смысла и полезности урока.»

**Активация и логика заполнения:**
> «Становится доступным только после заполнения основных учебных блоков (объяснение, примеры, практика, проверка). Автор может отредактировать или полностью переписать содержимое.»

**ИИ-поддержка:**
> «Допустима автоматизация. ИИ может по запросу автора сгенерировать черновик: краткий итог урока, вопросы для осмысления, ориентиры «что дальше». Все материалы ИИ подлежат обязательному редактированию и утверждению автором. ИИ не формирует выводы за ученика и не оценивает усвоение.»

**Предложения по доработке:**
> «1) Зафиксировать финальный блок как обязательный элемент завершения логики урока.
> 2) Реализовать генерацию черновика по кнопке.
> 3) Запретить автоматическое обновление блока без участия автора.
> 4) Добавить легенду с акцентом на осмысление, а не проверку.»

**Пример содержимого:**
> «Краткий итог; 2–4 вопроса для размышления; блок «что можно изучить дальше». ИИ используется только как инструмент синтеза.»
