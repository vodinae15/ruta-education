# Анализ требований заказчика по доработке системы блоков уроков

**Дата:** 20 января 2026 г.
**Проект:** Ruta Education
**Документ:** Техническое обоснование для коммерческого предложения

---

## 1. Краткое резюме

По итогам тестирования платформы с реальными авторами (фаундер, иностранный педагог, педагог, юрист) заказчик подготовил детальный анализ UX-проблем и предложения по доработке системы блоков уроков.

**Ключевое наблюдение заказчика:**
> Порядок заполнения автором (главный для конструктора) и порядок отображения ученику (главный для методики) — это два разных порядка, которые в текущей реализации не разделены, что приводит к путанице и снижению качества мета-блоков.

---

## 2. Сопоставление текущей архитектуры с требованиями

### 2.1. Текущая структура блоков

**Файл:** `lib/course-constructor-logic.ts` (строки 1-19)

```typescript
export interface CourseBlock {
  type:
    | "introduction"      // Дополнительный блок
    | "navigation"        // Навигация
    | "main_block_1"      // Основной блок 1
    | "intermediate_practice"  // Промежуточная практика
    | "main_block_2"      // Основной блок 2
    | "intermediate_test" // Промежуточный тест
    | "main_block_3"      // Основной блок 3
    | "conclusion"        // Итог и завершение
    | "bonus_support"     // Бонус и поддержка
}
```

### 2.2. Требуемая структура (9 блоков)

| № | Текущее название | Новое название | Статус | Тип изменения |
|---|-----------------|----------------|--------|---------------|
| 1 | `title` (поле) | Название урока/курса | Реализовано | Минимальные правки |
| 2 | `introduction` | **Как работать с этим уроком** | Требует переработки | Серьёзные изменения |
| 3 | `navigation` | Навигация | Требует переработки | Средние изменения |
| 4 | — | **Тезисы** (НОВЫЙ) | Не реализовано | Новая разработка |
| 5 | `main_block_1` | **Объяснение темы** | Требует переработки | Средние изменения |
| 6 | — | **Примеры** | Частично реализовано | Доработка |
| 7 | `intermediate_practice` | **Практика** | Требует переработки | Средние изменения |
| 8 | `intermediate_test` | **Проверка знаний** | Требует переработки | Средние изменения |
| 9 | `conclusion` | **Интеграция и завершение** | Частично реализовано | Доработка + AI |

## 3. Детальный анализ по блокам

### 3.1. Блок «Как работать с этим уроком» (приоритет: ВЫСОКИЙ)

**Текущее состояние:**
- Реализован как `introduction` (Дополнительный блок)
- Находится в линейном списке блоков
- Предлагается к заполнению первым
- Авторы не понимают его назначения

**Требуемые изменения:**

| Аспект | Текущее | Требуется |
|--------|---------|-----------|
| Название | "Дополнительный блок" | "Как работать с этим уроком" |
| Положение в конструкторе | Первый в списке | Отдельная зона, финальный этап |
| Порядок отображения ученику | Первый | Первый (без изменений) |
| Логика активации | Доступен сразу | Активируется после заполнения основного контента |
| Типы контента | Любые (теория, тесты) | Только инструктивный (текст, аудио, видео) |
| ИИ-поддержка | Нет | Шаблоны + генерация черновика |

**Технические задачи:**
1. Переименование типа блока и UI-меток
2. Вынос блока из линейного списка в отдельную зону интерфейса
3. Реализация логики активации (условие: осмысленный контент в основных блоках)
4. Ограничение доступных типов элементов
5. Создание методических шаблонов для AI-генерации
6. Интеграция AI для генерации черновика по шаблону
7. Визуальное отделение (контейнер/фон/рамка)

**Методические шаблоны (примеры):**
- "Управление ожиданиями" — что ученик получит/не получит
- "Снижение тревоги" — для сложных/объёмных тем
- "Объяснение формата" — как устроен урок

### 3.2. Блок «Навигация» (приоритет: СРЕДНИЙ)

**Текущее состояние:**
- Реализован как обязательный блок
- Заполняется вручную
- Дублирует структуру урока

**Требуемые изменения:**

| Аспект | Текущее | Требуется |
|--------|---------|-----------|
| Статус | Обязательный блок | Сервисный мета-элемент |
| Заполнение | Ручное | Автогенерация AI + редактирование |
| Положение | В линейном списке | Вне списка блоков |
| Обязательность | Да | Опциональный |
| Для коротких уроков | Показывается | Скрыт по умолчанию |

**Технические задачи:**
1. Вынос из линейного списка блоков
2. Реализация AI-генерации на основе структуры урока
3. Логика определения "короткого урока" (1 блок + ≤1 практика + ≤1 проверка)
4. Возможность редактирования/скрытия автором
5. Формат: только текст, без интерактива

### 3.3. Блок «Тезисы» (приоритет: ВЫСОКИЙ) — НОВЫЙ

**Текущее состояние:** Отсутствует

**Назначение:**
Вспомогательное поле для первичной выгрузки смыслов автором в свободной форме. Не отображается ученику.

**Требуемая реализация:**

| Аспект | Описание |
|--------|----------|
| Тип | Служебный элемент (не учебный) |
| Отображение автору | Слева от блока «Объяснение темы» |
| Отображение ученику | НЕ отображается |
| Ввод | Текст + аудио с транскрипцией |
| ИИ-поддержка | Группировка тезисов, выделение повторов (по запросу) |

**Технические задачи:**
1. Новый тип элемента `theses` в системе
2. UI: парное поле слева от основного блока
3. Интеграция с существующим API транскрипции `/api/transcribe`
4. Компонент записи/загрузки аудио
5. AI-помощник для анализа тезисов (опционально)
6. Поясняющая легенда с описанием назначения

## 4. Общие инфраструктурные изменения

### 4.1. Разделение порядков заполнения и отображения

**Проблема:** Текущая система не разделяет порядок работы автора и порядок показа ученику.

**Решение:**
```typescript
interface CourseBlock {
  // Новые поля
  authorOrder: number      // Порядок в конструкторе для автора
  studentOrder: number     // Порядок отображения ученику
  blockCategory: 'meta' | 'educational' | 'service'  // Категория блока
  activationCondition?: {
    type: 'content_threshold' | 'blocks_filled' | 'manual'
    params?: any
  }
}
```

### 4.2. Система состояний блоков

**Состояния для мета-блоков:**
1. `hidden` — скрыт/неактивен (по умолчанию)
2. `available` — доступен к активации
3. `editing` — редактируемый (после активации)
4. `published` — опубликован

### 4.3. Условия активации блоков

**Логика активации:**
- Блок "Как работать с этим уроком" — после появления осмысленного контента в основных блоках
- Навигация — после заполнения основных блоков (для длинных уроков)
- Интеграция и завершение — после заполнения объяснения + примеров + практики + проверки

**Критерий "осмысленного контента":**
- Минимальный порог по количеству символов (например, 500)
- Наличие заполненных элементов в ≥2 основных блоках

### 4.4. Поясняющие легенды

Для каждого блока добавить:
- Краткое описание назначения
- Ожидания от автора
- Ссылка на справочную страницу (опционально)

**Пример легенды для блока "Примеры":**
> Используйте примеры, чтобы показать, как идеи из объяснения работают на практике. Здесь ученик наблюдает, а не выполняет задания.

### 4.5. AI-интеграция

**Новые AI-функции:**
1. Генерация мета-блока "Как работать с этим уроком" по шаблону
2. Автогенерация навигации по структуре урока
3. Помощь в анализе тезисов (группировка, выделение повторов)
4. Генерация черновика проверочных вопросов
5. Генерация черновика завершения урока

**Принципы AI-интеграции:**
- Только по запросу автора (не автоматически)
- Не перезаписывает авторский текст
- Генерация в редактируемом виде
- Минимальный порог контента для генерации
