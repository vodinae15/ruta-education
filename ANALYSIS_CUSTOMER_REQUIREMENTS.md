# Анализ требований: Доработка системы блоков уроков

**Дата:** 20 января 2026 г.
**Проект:** Ruta Education

---

## Общая информация

### Источник требований
Таблица результатов тестирования платформы с 4 авторами разных профилей (фаундер, иностранный педагог, педагог, юрист).

### Ключевая проблема (цитата заказчика)
> «Ключевое наблюдение: разделяем 2 разных порядка:
> 1) порядок заполнения автором (главный для конструктора);
> 2) порядок отображения ученику (главный для методики).
> В текущей реализации эти порядки не разделены, что приводит к путанице и снижению качества мета-блока.»

## Блок 1: Название урока/курса

### Что пишет заказчик

**Назначение:**
> «Стартовое поле-якорь, фиксирующее намерение автора и тему разрабатываемого урока или курса. Помогает автору задать рамку работы и снизить неопределённость на старте.»

**Результаты тестирования:**
> «Авторы отмечают, что наличие названия помогает начать работу и понять, «про что» создаётся материал.»

**Предложения по доработке:** Нет (блок работает корректно)

### Как работает сейчас

1. Автор создаёт новый урок
2. Система сразу просит ввести название
3. Название отображается как заголовок урока
4. Автор может изменить название в любой момент

### Как будет работать

**Изменения не требуются.** Блок работает корректно по оценке заказчика.

### Оценка: 0 часов

## Блок 2: «Как работать с этим уроком» (сейчас: «Дополнительный блок»)

### Что пишет заказчик

**Назначение:**
> «Мета-блок для ученика, задающий рамку работы с уроком. Используется для:
> – управления ожиданиями ученика;
> – объяснения формата урока;
> – обозначения границ («о чём урок НЕ будет»);
> – снижения тревоги перед сложной/объёмной темой.»

**Результаты тестирования:**
> «Автор 1 (иностранный педагог): использует интуитивно, не до конца понимает статус блока, есть сомнение «что сюда писать».
> Автор 2 (педагог): блок полезен по смыслу, но создаёт путаницу из-за названия и равноправного положения среди учебных блоков.
> Автор 3 (юрист): не понимает назначение блока, игнорирует его заполнение.»

**Ключевое наблюдение:**
> «Автор не может качественно писать мета-блоки до тех пор, пока не закончил основной контент.»

**Предложения по доработке:**
> «1. Переименовать «Дополнительный блок» → «Как работать с этим уроком».
> 2. Выделить в отдельный тип (мета-блок), не равный учебным.
> 3. Ограничить типы элементов: только инструктивный контент (без практики, тестов).
> 4. Исключить из стартового сценария создания урока.
> 5. В опубликованном уроке отображать первым.
> 6. Визуально отделить от учебных блоков.
> 7. Блок становится доступным после заполнения основного контента + ИИ-поддержка с шаблонами.»

### Как работает сейчас

**Для автора:**
1. Автор открывает конструктор урока
2. Видит список блоков: «Дополнительный блок» стоит первым
3. Система предлагает заполнить его в первую очередь
4. Автор может добавить любые элементы: текст, видео, тесты, задания
5. Название «Дополнительный блок» не объясняет, что сюда писать
6. Блок выглядит как обычный учебный блок

**Для ученика:**
1. Ученик открывает урок
2. Видит «Дополнительный блок» первым
3. Блок никак визуально не отличается от учебного контента

**Проблемы:**
- Авторы не понимают назначение блока
- Заполняют его в начале, когда ещё не знают, о чём будет урок
- Или игнорируют совсем

### Как будет работать

**Для автора:**
1. Автор открывает конструктор урока
2. Блок «Как работать с этим уроком» **НЕ показывается** в начале
3. Автор работает с основным контентом (объяснение темы, примеры, практика)
4. **После того как автор заполнил основной контент** (набрал минимум 500 символов), появляется отдельная секция внизу страницы:

   ```
   ┌─────────────────────────────────────────────────┐
   │  📋 ФИНАЛЬНАЯ НАСТРОЙКА УРОКА                   │
   │                                                 │
   │  Как работать с этим уроком                     │
   │  ─────────────────────────────                  │
   │  Объясните ученику, как устроен урок           │
   │                                                 │
   │  Выберите шаблон:                              │
   │  ○ Управление ожиданиями                       │
   │  ○ Снижение тревоги перед сложной темой        │
   │  ○ Объяснение формата урока                    │
   │                                                 │
   │  [Сгенерировать черновик]  [Написать вручную]  │
   └─────────────────────────────────────────────────┘
   ```

5. Автор выбирает шаблон или пишет вручную
6. Если выбирает «Сгенерировать черновик» — ИИ создаёт текст на основе содержания урока
7. Автор редактирует текст как хочет
8. Доступны только: текст, видео, аудио (без тестов и заданий)

**Для ученика:**
1. Ученик открывает урок
2. **Первым видит** блок «Как работать с этим уроком»
3. Блок визуально отличается от учебного контента (другой фон, рамка)
4. Понимает, что это инструкция, а не учебный материал

### Оценка: 4-5 часов

| Задача | Часы |
|--------|------|
| Переименование и перенос блока в отдельную зону | 1.5 |
| Логика появления после заполнения контента | 0.5 |
| ИИ-генерация с 3 шаблонами | 1.5 |
| Визуальное оформление | 0.5 |
| Тестирование | 0.5 |

## Блок 3: Навигация

### Что пишет заказчик

**Назначение:**
> «Сервисный мета-элемент для ученика. Краткая карта урока: что будет происходить дальше и в какой логике.»

**Результаты тестирования:**
> «Фаундер: неочевидно назначение, ощущается дублирование структуры.
> Автор 2: блок отвлекает, создаёт ощущение навязанного шаблона.
> Автор 3: назначение неочевидно, заполнение игнорируется.
> Общее: навигация полезна только в сложных/длинных уроках.»

**Предложения по доработке:**
> «1) Убрать из линейного списка блоков.
> 2) Сделать опциональной.
> 3) Реализовать генерацию от структуры урока по запросу автора.
> 4) Формат: только текст, без интерактива.»

**Дополнительно:**
> «Короткий урок — 1 основной блок + не более 1 практики + не более 1 проверки. В коротких уроках навигация скрыта.»

### Как работает сейчас

**Для автора:**
1. Автор видит блок «Навигация» в списке обязательных блоков
2. Должен заполнить его вручную
3. Часто не понимает, что писать
4. Дублирует структуру урока или оставляет пустым

**Для ученика:**
1. Видит навигацию как обычный блок
2. Часто навигация бесполезна (особенно в коротких уроках)

### Как будет работать

**Для автора:**
1. Блок «Навигация» **НЕ показывается** в списке блоков
2. В секции «Финальная настройка урока» (рядом с «Как работать с уроком») появляется опция:

   ```
   ┌─────────────────────────────────────────────────┐
   │  🗺️ Навигация по уроку (опционально)            │
   │                                                 │
   │  Краткая карта урока для ученика               │
   │                                                 │
   │  [Сгенерировать автоматически]                 │
   │                                                 │
   │  Или оставить без навигации                    │
   └─────────────────────────────────────────────────┘
   ```

3. **Для коротких уроков** (1 блок объяснения + 1 практика) эта опция вообще не показывается
4. Если автор нажимает «Сгенерировать» — ИИ создаёт краткий список на основе структуры:
   ```
   В этом уроке:
   1. Разберём основные понятия темы
   2. Посмотрим примеры применения
   3. Попрактикуемся на заданиях
   4. Проверим, что запомнили
   ```
5. Автор может отредактировать или скрыть навигацию

**Для ученика:**
1. Если навигация включена — видит её после блока «Как работать с уроком»
2. Если урок короткий или автор отключил — навигации нет

### Оценка: 2-3 часа

| Задача | Часы |
|--------|------|
| Вынос из списка блоков | 0.5 |
| Логика определения короткого урока | 0.5 |
| ИИ-генерация по структуре | 1 |
| Кнопки управления (показать/скрыть) | 0.5 |
| Тестирование | 0.5 |

## Блок 4: Тезисы (НОВЫЙ)

### Что пишет заказчик

**Назначение:**
> «Вспомогательное поле для первичной выгрузки смыслов автором. Используется для фиксации основных мыслей, идей и логики темы в свободной форме. Тип контента: служебный (не учебный). Адресат: автор.»

**Результаты тестирования:**
> «Авторы отмечают затык на старте, страх формы и "переписывание" вместо осмысленного изложения. Голосовая фиксация мыслей воспринимается как более естественная.»

**Наблюдение:**
> «Авторам проще сначала зафиксировать смысл и логику темы (тезисно или голосом), а уже затем разворачивать объяснение.»

**Требования:**
> «Отображается слева от учебного блока. Поддерживает текстовый ввод и загрузку/запись аудио с транскрипцией. Не отображается ученику ни в каком виде.»

**ИИ-поддержка:**
> «ИИ по запросу автора может помочь осмыслить поток: предложить группировку тезисов, подсветить повторы. ИИ не переписывает тезисы.»

### Как работает сейчас

**Такого блока нет.** Автор сразу должен писать «готовый текст» в основной блок, что вызывает:
- Страх чистого листа
- Попытки сразу писать идеально
- Затык на старте

### Как будет работать

**Для автора:**
1. Автор открывает блок «Объяснение темы»
2. Слева от основного редактора видит поле «Тезисы»:

   ```
   ┌──────────────────────┬────────────────────────────────┐
   │  📝 ТЕЗИСЫ           │  ОБЪЯСНЕНИЕ ТЕМЫ               │
   │  (только для вас)    │                                │
   │                      │                                │
   │  Запишите ключевые   │  [Основной редактор контента]  │
   │  мысли в свободной   │                                │
   │  форме. Ученик это   │                                │
   │  не увидит.          │                                │
   │                      │                                │
   │  ┌────────────────┐  │                                │
   │  │                │  │                                │
   │  │  (текстовое    │  │                                │
   │  │   поле)        │  │                                │
   │  │                │  │                                │
   │  └────────────────┘  │                                │
   │                      │                                │
   │  🎤 Записать голосом │                                │
   │                      │                                │
   │  ✨ Помочь           │                                │
   │     структурировать  │                                │
   └──────────────────────┴────────────────────────────────┘
   ```

3. Автор может:
   - Набросать мысли текстом в свободной форме
   - Нажать «Записать голосом» и надиктовать мысли
   - Система автоматически превратит аудио в текст

4. Если тезисов много — автор нажимает «Помочь структурировать»:
   - ИИ группирует связанные мысли
   - Показывает повторы
   - Предлагает порядок изложения
   - **Не переписывает** текст автора, только помогает организовать

5. Глядя на тезисы слева, автор пишет основной текст справа

**Для ученика:**
- Тезисы **НЕ отображаются** — это внутренний инструмент автора

### Оценка: 4-5 часов

| Задача | Часы |
|--------|------|
| Поле тезисов рядом с основным блоком | 1.5 |
| Запись голоса + транскрипция | 1.5 |
| ИИ-помощник для структурирования | 1 |
| Тестирование | 0.5 |

## Блок 5: Объяснение темы (сейчас: «Основной блок»)

### Что пишет заказчик

**Назначение:**
> «Основной учебный блок для формирования у ученика целостного понимания темы. Используется для объяснения ключевых понятий, логики и взаимосвязей без перехода к применению знаний.»

**Результаты тестирования:**
> «Авторы перегружаются на старте и пытаются сразу писать "готовый текст". Без чётких границ блока авторы смешивают объяснение, примеры и практику.»

**Предложения по доработке:**
> «1) Добавить поясняющую легенду с ожиданиями от автора.
> 2) Разместить ссылку на справочную страницу с рекомендациями.
> 3) Зафиксировать парность с блоком «Тезисы».
> 4) ИИ в блоке отсутствует — зона чистого авторского объяснения.»

### Как работает сейчас

**Для автора:**
1. Видит блок с названием «Основной блок 1»
2. Не понимает, что именно сюда писать
3. Может добавить что угодно: теорию, примеры, задания
4. Нет подсказок о границах блока

### Как будет работать

**Для автора:**
1. Блок называется «Объяснение темы»
2. Под заголовком — поясняющая легенда:

   ```
   ┌─────────────────────────────────────────────────────┐
   │  📖 ОБЪЯСНЕНИЕ ТЕМЫ                                 │
   │                                                     │
   │  ℹ️ Здесь вы объясняете тему: ключевые понятия,     │
   │     логика, взаимосвязи.                           │
   │     Примеры и задания — в отдельных блоках.        │
   │                                                     │
   │     📚 Советы по объяснению сложных тем →          │
   │                                                     │
   │  ┌───────────────────────────────────────────────┐ │
   │  │                                               │ │
   │  │  [Редактор контента]                          │ │
   │  │                                               │ │
   │  └───────────────────────────────────────────────┘ │
   └─────────────────────────────────────────────────────┘
   ```

3. Слева — поле «Тезисы» (см. Блок 4)
4. Ссылка «Советы по объяснению» открывает справочную страницу
5. **ИИ-подсказки отключены** — это зона чистого авторского объяснения

**Для ученика:**
- Видит понятный заголовок «Объяснение темы» вместо «Основной блок 1»

### Оценка: 1-1.5 часа

| Задача | Часы |
|--------|------|
| Переименование блока | 0.3 |
| Добавление легенды | 0.5 |
| Справочная страница (контент) | 0.5 |
| Тестирование | 0.2 |

## Блок 6: Примеры

### Что пишет заказчик

**Назначение:**
> «Учебный блок для демонстрации применения идей. Используется для перехода от абстрактного понимания к конкретному представлению «как это работает». Ученик наблюдает, а не выполняет задания.»

**Результаты тестирования:**
> «Авторы либо не добавляют примеры вовсе, либо перегружают ими основной текст. Ученики испытывают трудности при переходе к практике без предварительных примеров.»

**Предложения по доработке:**
> «1) Зафиксировать статус примеров как демонстрационного блока.
> 2) Запретить размещение заданий и тестов внутри блока.
> 3) Добавить легенду.»

**Легенда от заказчика:**
> «Используйте примеры, чтобы показать, как идеи из объяснения работают на практике. Здесь ученик наблюдает, а не выполняет задания.»

### Как работает сейчас

**Для автора:**
1. Примеры можно добавить в любой блок
2. Нет отдельного блока для примеров
3. Автор может смешать примеры с заданиями

### Как будет работать

**Для автора:**
1. После «Объяснения темы» идёт отдельный блок «Примеры»
2. Блок опциональный — можно пропустить
3. Легенда объясняет назначение:

   ```
   ┌─────────────────────────────────────────────────────┐
   │  💡 ПРИМЕРЫ                                         │
   │                                                     │
   │  ℹ️ Покажите, как идеи из объяснения работают на   │
   │     практике. Здесь ученик наблюдает, а не         │
   │     выполняет задания.                             │
   │                                                     │
   │     📚 Как сделать примеры наглядными →            │
   │                                                     │
   │  Доступные элементы: текст, изображения, видео     │
   │  ⛔ Задания и тесты недоступны в этом блоке        │
   └─────────────────────────────────────────────────────┘
   ```

4. При попытке добавить задание — система показывает:
   «Задания добавляются в блок "Практика"»

**Для ученика:**
- Понимает, что это демонстрация, а не задание

### Оценка: 1 час

| Задача | Часы |
|--------|------|
| Настройка блока + легенда | 0.5 |
| Ограничение типов элементов | 0.3 |
| Тестирование | 0.2 |

## Блок 7: Практика

### Что пишет заказчик

**Назначение:**
> «Учебный блок для безопасного применения знаний учеником. Переход от понимания к самостоятельному действию без оценивания.»

**Результаты тестирования:**
> «Авторы часто используют тесты как практику. Ученики воспринимают задания как оценку и боятся ошибаться.»

**Предложения по доработке:**
> «1) Зафиксировать практику как неоценочную.
> 2) Поддержать несколько форм практики.
> 3) Добавить легенду.»

**Поддерживаемые формы:**
> «открытый ответ; задание на применение; кейс/сценарий; выбор с объяснением; сопоставление/классификация.»

### Как работает сейчас

**Для автора:**
1. Блок называется «Промежуточная практика»
2. Нет указания, что это неоценочный блок
3. Ограниченные форматы заданий

**Для ученика:**
1. Задания могут восприниматься как тест
2. Ошибки подсвечиваются как «неправильно»
3. Страх ошибиться

### Как будет работать

**Для автора:**
1. Блок называется «Практика»
2. Легенда объясняет подход:

   ```
   ┌─────────────────────────────────────────────────────┐
   │  🎯 ПРАКТИКА                                        │
   │                                                     │
   │  ℹ️ Здесь ученик пробует применить знания.          │
   │     Ошибки — часть обучения, они не влияют         │
   │     на оценку.                                     │
   │                                                     │
   │     📚 Форматы практики и как выбрать →            │
   │                                                     │
   │  Выберите формат:                                  │
   │  ○ Открытый ответ                                  │
   │  ○ Задание на применение                           │
   │  ○ Кейс/сценарий                                   │
   │  ○ Выбор с объяснением                             │
   │  ○ Сопоставление                                   │
   └─────────────────────────────────────────────────────┘
   ```

3. 5 форматов практики на выбор

**Для ученика:**
1. Понимает, что это тренировка, не экзамен
2. При ошибке видит нейтральную обратную связь:
   - ❌ Было: «Неправильно!»
   - ✅ Стало: «Интересный подход! Рассмотрите также такой вариант...»
3. Нет красных крестиков и оценок

### Оценка: 2-3 часа

| Задача | Часы |
|--------|------|
| Переименование + легенда | 0.5 |
| 5 форматов практики | 1.5 |
| Изменение обратной связи (нейтральная) | 0.5 |
| Тестирование | 0.5 |

## Блок 8: Проверка знаний

### Что пишет заказчик

**Назначение:**
> «Учебный блок для осознанного извлечения и закрепления изученного. Чтобы ученик понял, что именно он усвоил и где остались пробелы. Тип: диагностический (не оценочный).»

**Результаты тестирования:**
> «Ученики испытывают тревожность при проверке. Авторы часто используют тесты с единственно верным ответом, что снижает глубину усвоения.»

**Предложения по доработке:**
> «1) Зафиксировать проверку как диагностическую, не оценочную.
> 2) Результаты показывать в формате "что получилось, где стоит повторить".
> 3) ИИ может предложить черновик вопросов по содержанию урока.»

**Поддерживаемые формы:**
> «тесты с объяснением; открытые вопросы на понимание; задания на извлечение знаний; рефлексивные вопросы.»

### Как работает сейчас

**Для автора:**
1. Блок называется «Промежуточный тест»
2. Создаёт вопросы вручную
3. Нет ИИ-помощника

**Для ученика:**
1. Название «тест» вызывает тревогу
2. Результаты показываются как оценка: «3 из 5 правильно»
3. Ощущение экзамена

### Как будет работать

**Для автора:**
1. Блок называется «Проверка знаний»
2. Легенда:

   ```
   ┌─────────────────────────────────────────────────────┐
   │  🔍 ПРОВЕРКА ЗНАНИЙ                                 │
   │                                                     │
   │  ℹ️ Помогите ученику понять, что он усвоил.         │
   │     Это не экзамен, а инструмент самопроверки.     │
   │                                                     │
   │  [✨ Сгенерировать вопросы по уроку]               │
   │                                                     │
   │  Или создайте вопросы вручную                      │
   └─────────────────────────────────────────────────────┘
   ```

3. Кнопка «Сгенерировать вопросы» — ИИ создаёт 3-5 вопросов на основе урока
4. Автор редактирует/добавляет/удаляет вопросы

**Для ученика:**
1. Название не пугает — «Проверка знаний», не «Тест»
2. После прохождения видит диагностику, а не оценку:

   ```
   ┌─────────────────────────────────────────────────────┐
   │  📊 РЕЗУЛЬТАТЫ САМОПРОВЕРКИ                         │
   │                                                     │
   │  ✅ Вы хорошо усвоили:                              │
   │     • Основные понятия темы                        │
   │     • Принципы работы механизма                    │
   │                                                     │
   │  📖 Стоит повторить:                               │
   │     • Различия между типами А и Б                  │
   │       [Вернуться к разделу →]                      │
   └─────────────────────────────────────────────────────┘
   ```

3. Нет оценок, процентов, красных крестиков
4. Есть ссылки на разделы для повторения

### Оценка: 3-4 часа

| Задача | Часы |
|--------|------|
| Переименование + легенда | 0.5 |
| ИИ-генерация вопросов | 1.5 |
| Диагностический формат результатов | 1 |
| Тестирование | 0.5 |

## Блок 9: Интеграция и завершение

### Что пишет заказчик

**Назначение:**
> «Финальный блок для осмысления, закрепления и интеграции изученного. Подведение итогов, формирование целостной картины и ощущение завершённости урока.»

**Результаты тестирования:**
> «При отсутствии финального блока ученики хуже осознают, что именно усвоили, и воспринимают урок как незавершённый фрагмент.»

**Предложения по доработке:**
> «1) Зафиксировать как обязательный элемент завершения.
> 2) ИИ может сгенерировать черновик: итог + вопросы для размышления + что дальше.
> 3) Легенда с акцентом на осмысление, а не проверку.»

**Пример содержимого:**
> «Краткий итог; 2–4 вопроса для размышления; блок «что можно изучить дальше».»

### Как работает сейчас

**Для автора:**
1. Блок «Итог и завершение» доступен сразу
2. Нет структуры — просто пустое поле
3. Часто заполняется формально или пропускается

**Для ученика:**
1. Урок может обрываться без логического завершения
2. Нет ощущения целостности

### Как будет работать

**Для автора:**
1. Блок появляется в секции «Финальная настройка» **после заполнения основных блоков**
2. Структурированная форма:

   ```
   ┌─────────────────────────────────────────────────────┐
   │  🎓 ИНТЕГРАЦИЯ И ЗАВЕРШЕНИЕ                         │
   │                                                     │
   │  ℹ️ Помогите ученику осмыслить урок.                │
   │                                                     │
   │  [✨ Сгенерировать черновик]                        │
   │                                                     │
   │  ────────────────────────────────────               │
   │                                                     │
   │  Краткий итог урока:                               │
   │  ┌─────────────────────────────────────────────┐   │
   │  │                                             │   │
   │  └─────────────────────────────────────────────┘   │
   │                                                     │
   │  Вопросы для размышления (2-4):                    │
   │  1. [                                         ]    │
   │  2. [                                         ]    │
   │  3. [                                         ]    │
   │  [+ Добавить вопрос]                              │
   │                                                     │
   │  Что изучить дальше:                              │
   │  ┌─────────────────────────────────────────────┐   │
   │  │                                             │   │
   │  └─────────────────────────────────────────────┘   │
   └─────────────────────────────────────────────────────┘
   ```

3. Кнопка «Сгенерировать черновик» — ИИ заполняет все поля на основе урока
4. Автор редактирует как хочет

**Для ученика:**
1. Урок заканчивается логично
2. Видит краткий итог — что главное запомнить
3. Вопросы для размышления — помогают осмыслить
4. «Что дальше» — понимает куда двигаться

### Оценка: 2-3 часа

## Общие изменения

### Разделение порядка заполнения и отображения

**Сейчас:** Автор видит блоки в том же порядке, в каком их увидит ученик.

**Станет:**
- **Для автора:** Сначала основной контент → потом мета-блоки
- **Для ученика:** Сначала мета-блоки → потом основной контент

```
ПОРЯДОК ДЛЯ АВТОРА:              ПОРЯДОК ДЛЯ УЧЕНИКА:
1. Объяснение темы          →    1. Как работать с уроком
2. Примеры                  →    2. Навигация (если есть)
3. Практика                 →    3. Объяснение темы
4. Проверка знаний          →    4. Примеры
   ─────────────────────         5. Практика
   Финальная настройка:          6. Проверка знаний
5. Как работать с уроком    →    7. Интеграция и завершение
6. Навигация (опционально)
7. Интеграция и завершение
```

### Миграция существующих курсов

Все существующие курсы продолжат работать. Новые названия блоков применятся автоматически:
- «Дополнительный блок» → «Как работать с этим уроком»
- «Основной блок» → «Объяснение темы»
- «Промежуточная практика» → «Практика»
- «Промежуточный тест» → «Проверка знаний»
- «Итог и завершение» → «Интеграция и завершение»

### Справочные страницы

Создаём 5 страниц с рекомендациями для авторов:
1. Как объяснять сложные темы
2. Как делать примеры наглядными
3. Форматы практики и как выбрать
4. Как создавать проверку знаний
5. Как завершать урок
