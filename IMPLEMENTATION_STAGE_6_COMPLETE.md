### 6.1. Улучшена валидация адаптаций для трехслойной структуры

**Файл**: `app/api/ai-adaptation/route.ts`

**Функциональность**:
- Детальная валидация всех трех слоев каждого блока
- Проверка наличия подводки (intro), контента (content) и адаптированного элемента (adaptation)
- Валидация типов данных и минимальных требований к длине текста
- Детальные сообщения об ошибках с указанием конкретного блока и поля

**Новый интерфейс ValidationError**:
```typescript
interface ValidationError {
  block: string
  field: string
  message: string
}
```

**Проверки валидации**:
- **Подводка (intro)**:
  - Наличие поля `intro`
  - Наличие и валидность `intro.text` (строка, не пустая)
  - Минимальная длина `intro.text` (10 символов)
  
- **Контент (content)**:
  - Наличие поля `content`
  - Наличие и валидность `content.title` (строка, не пустая)
  - Наличие и валидность `content.text` (строка, не пустая)
  - Минимальная длина `content.text` (50 символов)
  - Тип `content.type` должен быть 'text'
  
- **Адаптированный элемент (adaptation)**:
  - Наличие поля `adaptation`
  - Тип `adaptation.type` должен быть 'visual', 'auditory' или 'kinesthetic'
  - Наличие поля `adaptation.element`
  - Наличие `adaptation.element.type`
  - Наличие `adaptation.element.data`
  - Наличие и валидность `adaptation.element.description` (строка)

## 🚀 Как использовать

### 1. Валидация адаптированного контента

```typescript
const validation = validateAdaptationContent(adaptedContent)
if (!validation.valid) {
  // Обработка ошибок валидации
  validation.errors.forEach(error => {
    console.error(`Блок ${error.block}: ${error.field} - ${error.message}`)
  })
}
```

### 2. Обработка ошибок

```typescript
try {
  const response = await fetch('/api/ai-adaptation', {
    method: 'POST',
    body: JSON.stringify({
      lessonContent,
      studentType: 'visual',
      lessonId: 'lesson-id',
      saveToDatabase: true,
      forceRegenerate: false
    })
  })
  
  if (!response.ok) {
    const error = await response.json()
    console.error('Error:', error.error)
    if (error.validationErrors) {
      error.validationErrors.forEach((err: ValidationError) => {
        console.error(`Block ${err.block}: ${err.field} - ${err.message}`)
      })
    }
  }
} catch (error) {
  console.error('Error:', error)
}
```

### 3. Проверка материалов урока

```typescript
const { data: materialsAnalysis } = await supabase.rpc('analyze_lesson_materials', {
  lesson_id_param: lessonId
})

console.log('Materials:', {
  has_audio: materialsAnalysis.has_audio,
  has_video: materialsAnalysis.has_video,
  has_images: materialsAnalysis.has_images,
  has_diagrams: materialsAnalysis.has_diagrams,
  has_practice: materialsAnalysis.has_practice
})
```

### 4. Сохранение адаптации

```typescript
// Автоматическое сохранение при успешной генерации
const response = await fetch('/api/ai-adaptation', {
  method: 'POST',
  body: JSON.stringify({
    lessonContent,
    studentType: 'visual',
    lessonId: 'lesson-id',
    saveToDatabase: true, // По умолчанию true
    forceRegenerate: false
  })
})

if (response.ok) {
  const result = await response.json()
  console.log('Adaptation saved:', result.savedToDatabase)
}
```

## ⚠️ Важные замечания

1. **Валидация**:
   - Валидация выполняется после парсинга JSON
   - Ошибки валидации не блокируют сохранение, но возвращают ошибку клиенту
   - Детальные сообщения об ошибках помогают отладке

2. **Обработка ошибок**:
   - Все ошибки логируются с детальной информацией
   - Статус ошибки обновляется в метаданных
   - Клиент получает детальную информацию об ошибках

3. **Проверка материалов**:
   - Анализ материалов не блокирует процесс адаптации
   - Результаты анализа логируются для отладки
   - Ошибки анализа не критичны

4. **Сохранение в БД**:
   - Сохранение выполняется автоматически при успешной генерации
   - Версия увеличивается при обновлении
   - Метаданные обновляются при любом изменении статуса

5. **Права доступа**:
   - Только автор или соавтор могут генерировать адаптации
   - Проверка прав выполняется перед началом адаптации
   - Неавторизованные пользователи получают ошибку 401

6. **Параметры запроса**:
   - `max_tokens` увеличен до 8000 для более длинных адаптаций
   - `response_format` гарантирует JSON-ответ
   - Системный промпт явно указывает структуру блоков
