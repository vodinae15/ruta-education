# Инструкция по развертыванию Next.js проекта на сервере

## ЧАСТЬ 1: Подготовка проекта (на локальной машине)

### 1.1 Структура переменных окружения

При разработке на localhost важно правильно настроить окружение:

**Для разработки:** `.env.local`
```env
# Supabase (одинаковые для dev и prod)
NEXT_PUBLIC_SUPABASE_URL="https://your-project.supabase.co"
NEXT_PUBLIC_SUPABASE_ANON_KEY="ваш_anon_ключ"
SUPABASE_SERVICE_ROLE_KEY="ваш_service_role_ключ"

# URLs для разработки
NEXT_PUBLIC_APP_URL=http://localhost:3000
NEXT_PUBLIC_DEV_SUPABASE_REDIRECT_URL=http://localhost:3000/dashboard

# Development settings
NODE_ENV=development

# AI Services (опционально)
OPENROUTER_API_KEY="ваш_ключ"
CLAUDE_API_KEY="ваш_ключ"
```

**Для production:** `.env.production`
```env
# Supabase (те же что и в dev)
NEXT_PUBLIC_SUPABASE_URL="https://your-project.supabase.co"
NEXT_PUBLIC_SUPABASE_ANON_KEY="ваш_anon_ключ"
SUPABASE_SERVICE_ROLE_KEY="ваш_service_role_ключ"

# URLs для production (IP или домен)
NEXT_PUBLIC_APP_URL=http://93.189.229.85
NEXT_PUBLIC_DEV_SUPABASE_REDIRECT_URL=http://93.189.229.85/dashboard

# Production settings
NODE_ENV=production
PORT=3000

# AI Services (опционально)
OPENROUTER_API_KEY="ваш_ключ"
CLAUDE_API_KEY="ваш_ключ"
```

### 1.2 Важные моменты при подготовке:

**1. Проверь .gitignore:**
```
.env
.env.local
.env.production
.next
node_modules
```

**2. Не коммить секреты в Git:**
- API ключи должны быть только в `.env` файлах
- `.env` файлы должны быть в `.gitignore`
- Создай `.env.example` с пустыми значениями для документации

**3. Настрой Supabase:**
- Создай проект в Supabase
- Скопируй URL и ключи из Settings → API
- Выполни миграцию БД (SQL скрипт)

**4. Протестируй локально:**
```bash
# Development
npm run dev

# Production build (локально)
npm run build
npm start
```

## ШАГ 6: Запуск приложения через PM2

### 6.1 Создай конфигурацию PM2

```bash
cat > ecosystem.config.js << 'EOF'
require('dotenv').config({ path: '.env.production' })

module.exports = {
  apps: [{
    name: 'ruta-education',
    script: 'node_modules/next/dist/bin/next',
    args: 'start',
    cwd: '/var/www/ruta-education',
    instances: 2,
    exec_mode: 'cluster',
    env_file: '.env.production',
    env: {
      NODE_ENV: 'production',
      PORT: 3000
    }
  }]
}
EOF
```

### 6.2 Запуск приложения

```bash
# Запуск
pm2 start ecosystem.config.js

# Сохранение конфигурации
pm2 save

# Автозапуск при перезагрузке сервера
pm2 startup
```

После `pm2 startup` может появиться команда для выполнения - выполни её.

### 6.3 Проверка статуса

```bash
pm2 status
```

Должно быть:
```
┌────┬────────────────────┬──────────┬──────┬───────────┬──────────┬──────────┐
│ id │ name               │ mode     │ ↺    │ status    │ cpu      │ memory   │
├────┼────────────────────┼──────────┼──────┼───────────┼──────────┼──────────┤
│ 0  │ ruta-education     │ cluster  │ 0    │ online    │ 0%       │ 77.7mb   │
│ 1  │ ruta-education     │ cluster  │ 0    │ online    │ 0%       │ 78.2mb   │
└────┴────────────────────┴──────────┴──────┴───────────┴──────────┴──────────┘
```

**Status должен быть `online`!**

## ШАГ 7: Настройка Nginx

### 7.1 Создай конфигурацию для сайта

```bash
cat > /etc/nginx/sites-available/ruta-education << 'EOF'
server {
    listen 80;
    server_name 93.189.229.85;

    location / {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_cache_bypass $http_upgrade;
    }
}
EOF
```

**Замени `server_name 93.189.229.85;` на свой IP или домен!**

### 7.2 Активируй конфигурацию

```bash
# Создай символическую ссылку
ln -s /etc/nginx/sites-available/ruta-education /etc/nginx/sites-enabled/

# Удали дефолтную конфигурацию
rm -f /etc/nginx/sites-enabled/default

# Проверь конфигурацию
nginx -t
```

Должно быть:
```
nginx: configuration file /etc/nginx/nginx.conf test is successful
```

### 7.3 Перезапусти Nginx

```bash
systemctl restart nginx
```

Проверь статус:
```bash
systemctl status nginx
```

## ЧАСТЬ 4: Подключение домена (опционально)

### 4.1 Настройка DNS

В настройках домена (например на reg.ru, namecheap и т.д.):

Добавь A-запись:
```
Type: A
Name: @ (или www)
Value: 93.189.229.85
TTL: 3600
```

Подожди 10-60 минут пока DNS обновится.

### 4.2 Обновление Nginx конфигурации

```bash
nano /etc/nginx/sites-available/ruta-education
```

Измени `server_name`:
```nginx
server {
    listen 80;
    server_name yourdomain.com www.yourdomain.com;
    # ... остальное без изменений
}
```

Перезапусти:
```bash
nginx -t
systemctl restart nginx
```

### 4.3 Обновление .env.production

```bash
cd /var/www/ruta-education
nano .env.production
```

Измени URLs:
```env
NEXT_PUBLIC_APP_URL=https://yourdomain.com
NEXT_PUBLIC_DEV_SUPABASE_REDIRECT_URL=https://yourdomain.com/dashboard
```

Пересобери и перезапусти:
```bash
npm run build
pm2 restart ruta-education
```

### 4.4 Установка SSL сертификата (HTTPS)

```bash
# Установка Certbot
apt install -y certbot python3-certbot-nginx

# Получение сертификата
certbot --nginx -d yourdomain.com -d www.yourdomain.com
```

Следуй инструкциям Certbot:
- Введи email
- Согласись с условиями (Y)
- Выбери редирект HTTP → HTTPS (2)

Автообновление сертификата настроится автоматически.

## ЧАСТЬ 5: Решение проблем

### Приложение не запускается

Проверь логи:
```bash
pm2 logs ruta-education --lines 100
```

Частые проблемы:
- Ошибка в .env.production (неправильные ключи)
- Порт 3000 занят
- Недостаточно памяти

### Ошибка подключения к Supabase

Проверь:
1. Правильность URL и ключей в .env.production
2. Проект Supabase активен
3. Выполнена миграция БД

### Nginx не запускается

```bash
# Проверь конфигурацию
nginx -t

# Проверь логи
tail -f /var/log/nginx/error.log

# Проверь не занят ли порт 80
netstat -tulpn | grep :80
```

### Приложение работает, но страницы не открываются

Проверь:
1. PM2 статус: `pm2 status`
2. Nginx статус: `systemctl status nginx`
3. Порт 3000 слушается: `netstat -tulpn | grep :3000`
4. Файрвол не блокирует порт 80: `ufw status`
