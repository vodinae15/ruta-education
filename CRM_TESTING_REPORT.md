## ⚠️ НАЙДЕННЫЕ ПРОБЛЕМЫ И РЕКОМЕНДАЦИИ

### 🔴 КРИТИЧЕСКИЕ ПРОБЛЕМЫ

#### 1. Проблема с пустым `courseId` в заметках
**Файл:** `components/ui/student-statistics-table.tsx:212`
```typescript
courseId={courseId || ""}
```

**Проблема:** Если `courseId` равен `null`, передается пустая строка `""`, что может вызвать ошибку в API `/api/student-notes`, который требует валидный `courseId`.

**Решение:** Не открывать модальное окно заметок, если `courseId` не выбран:
```typescript
{notesOpen && selectedStudentId && courseId && (
  <StudentNotesManager
    studentId={selectedStudentId}
    courseId={courseId}
    isOpen={notesOpen}
    onClose={handleCloseNotes}
  />
)}
```

**Статус:** Требует исправления

---

### 🟡 СРЕДНИЕ ПРОБЛЕМЫ

#### 2. Двойная загрузка данных при инициализации
**Файл:** `app/students/page.tsx:49-71`

**Проблема:** При первой загрузке страницы вызываются оба `useEffect`:
- Первый вызывает `loadData()` (загружает курсы и учеников)
- Второй вызывает `loadStudents()` (загружает только учеников)

Это приводит к двойному запросу к API при первой загрузке.

**Решение:** Убрать вызов `loadStudents()` из второго `useEffect` при первой загрузке, или добавить проверку:
```typescript
useEffect(() => {
  if (user && !authLoading && !loading) {
    loadStudents()
  }
}, [selectedCourseId, user, authLoading])
```

**Статус:** Рекомендуется исправить для оптимизации

#### 3. Отсутствие валидации email при добавлении ученика
**Файл:** `app/students/page.tsx:137`

**Проблема:** Проверяется только наличие текста, но не валидность email-адреса.

**Решение:** Добавить валидацию email:
```typescript
const emailRegex = /^[^\s@]+@[^\s@]+\.[^\s@]+$/
if (!emailRegex.test(email.trim())) {
  toast({
    title: "Ошибка",
    description: "Введите корректный email адрес",
    variant: "destructive",
  })
  return
}
```

**Статус:** Рекомендуется добавить

#### 4. Отсутствие обработки случая, когда курс не выбран при удалении доступа
**Файл:** `app/students/page.tsx:204`

**Проблема:** Если `selectedCourseId` равен `null`, показывается ошибка, но это может быть неочевидно для пользователя, когда он просматривает "Все курсы".

**Решение:** При просмотре всех курсов нужно либо:
- Запретить удаление доступа (показать сообщение "Выберите курс")
- Или передавать `courseId` из данных ученика в таблице

**Статус:** Требует доработки логики
