### 9.2. Создание скрипта для проверки существующих адаптаций

**Файл**: `scripts/26-check-existing-adaptations.sql`

**Функциональность**:
- Функция `get_adaptations_statistics()` - возвращает статистику по адаптациям
- Функция `get_lessons_without_adaptations()` - возвращает список уроков без адаптаций
- Функция `get_lessons_with_incomplete_adaptations()` - возвращает уроки с неполными адаптациями
- Функция `validate_adaptation_structure()` - валидирует структуру адаптации
- Функция `get_adaptations_with_errors()` - возвращает адаптации с ошибками валидации

**Использование**:
```sql
-- Получить статистику по адаптациям
SELECT * FROM get_adaptations_statistics();

-- Получить уроки без адаптаций
SELECT * FROM get_lessons_without_adaptations();

-- Получить уроки с неполными адаптациями
SELECT * FROM get_lessons_with_incomplete_adaptations();

-- Валидировать адаптацию
SELECT * FROM validate_adaptation_structure('adaptation_id');

-- Получить адаптации с ошибками
SELECT * FROM get_adaptations_with_errors();
```

### 9.3. Создание скрипта для валидации мигрированных данных


### 9.4. Создание API endpoint для массовой генерации адаптаций

**Файл**: `app/api/adaptations/generate-batch/route.ts`

**Функциональность**:
- Генерирует адаптации для нескольких уроков одновременно
- Поддерживает генерацию для указанных уроков или всех уроков курса
- Поддерживает генерацию для указанных типов адаптации или всех типов
- Поддерживает принудительную регенерацию существующих адаптаций
- Обновляет статус генерации в метаданных
- Обрабатывает ошибки и обновляет статус ошибок в метаданных

**Параметры запроса**:
```typescript
{
  courseId?: string        // ID курса для генерации адаптаций
  lessonIds?: string[]     // Массив ID уроков для генерации адаптаций
  adaptationTypes?: string[] // Массив типов адаптации (по умолчанию все типы)
  forceRegenerate?: boolean  // Перегенерировать существующие адаптации
}
```

**Использование**:
```typescript
// Генерация адаптаций для всех уроков курса
POST /api/adaptations/generate-batch
{
  "courseId": "course-id",
  "adaptationTypes": ["visual", "auditory", "kinesthetic"],
  "forceRegenerate": false
}

// Генерация адаптаций для указанных уроков
POST /api/adaptations/generate-batch
{
  "lessonIds": ["lesson-id-1", "lesson-id-2"],
  "adaptationTypes": ["visual"],
  "forceRegenerate": true
}
```

**Ответ**:
```typescript
{
  success: boolean
  message: string
  results: Array<{
    lessonId: string
    lessonTitle: string
    adaptationType: string
    success: boolean
    error?: string
  }>
  statistics: {
    total: number
    success: number
    errors: number
    lessonsProcessed: number
    adaptationsPerLesson: number
  }
}
```

### 9.5. Добавление обработки ошибок при миграции

**Файл**: `app/api/adaptations/generate-batch/route.ts`

**Функциональность**:
- Обновление статуса генерации в метаданных перед началом генерации
- Обновление статуса ошибки в метаданных при ошибке генерации
- Обработка ошибок HTTP запросов
- Обработка ошибок парсинга JSON
- Обработка ошибок валидации
- Обработка ошибок сохранения в БД
- Логирование всех ошибок

**Обработка ошибок**:
- Ошибки аутентификации: возвращается 401
- Ошибки доступа: возвращается 403
- Ошибки загрузки уроков: возвращается 500
- Ошибки генерации адаптации: сохраняется в метаданных, продолжается генерация для других уроков
- Ошибки обновления метаданных: логируются, не прерывают выполнение

## 🔍 Структура реализации

### 1. Проверка существующих адаптаций

```sql
-- Статистика по адаптациям
SELECT * FROM get_adaptations_statistics();
-- Возвращает:
-- - total_lessons: общее количество уроков
-- - lessons_with_adaptations: количество уроков с адаптациями
-- - lessons_without_adaptations: количество уроков без адаптаций
-- - total_adaptations: общее количество адаптаций
-- - visual_adaptations: количество визуальных адаптаций
-- - auditory_adaptations: количество аудиальных адаптаций
-- - kinesthetic_adaptations: количество кинестетических адаптаций
-- - published_adaptations: количество опубликованных адаптаций
-- - pending_adaptations: количество ожидающих адаптаций
-- - generated_adaptations: количество сгенерированных адаптаций
-- - edited_adaptations: количество отредактированных адаптаций

-- Уроки без адаптаций
SELECT * FROM get_lessons_without_adaptations();
-- Возвращает:
-- - lesson_id: ID урока
-- - lesson_title: название урока
-- - course_id: ID курса
-- - course_title: название курса
-- - author_id: ID автора
-- - blocks_count: количество блоков
-- - has_elements: есть ли элементы в блоках

-- Уроки с неполными адаптациями
SELECT * FROM get_lessons_with_incomplete_adaptations();
-- Возвращает:
-- - lesson_id: ID урока
-- - lesson_title: название урока
-- - course_id: ID курса
-- - course_title: название курса
-- - existing_types: массив существующих типов адаптаций
-- - missing_types: массив отсутствующих типов адаптаций
```

### 2. Валидация мигрированных данных

```sql
-- Валидация всех адаптаций
SELECT * FROM validate_all_adaptations();
-- Возвращает:
-- - lesson_id: ID урока
-- - lesson_title: название урока
-- - course_id: ID курса
-- - course_title: название курса
-- - adaptation_type: тип адаптации
-- - status: статус адаптации
-- - is_valid: валидна ли адаптация
-- - errors: массив ошибок
-- - block1_valid: валиден ли блок 1
-- - block2_valid: валиден ли блок 2
-- - block3_valid: валиден ли блок 3
-- - block4_valid: валиден ли блок 4
-- - block5_valid: валиден ли блок 5

-- Статистика валидации
SELECT * FROM get_validation_statistics();
-- Возвращает:
-- - total_adaptations: общее количество адаптаций
-- - valid_adaptations: количество валидных адаптаций
-- - invalid_adaptations: количество невалидных адаптаций
-- - adaptations_with_errors: количество адаптаций с ошибками
-- - block1_errors: количество ошибок в блоке 1
-- - block2_errors: количество ошибок в блоке 2
-- - block3_errors: количество ошибок в блоке 3
-- - block4_errors: количество ошибок в блоке 4
-- - block5_errors: количество ошибок в блоке 5
```

## ⚠️ Важные замечания

1. **Проверка существующих адаптаций**:
   - Функции проверки используют существующие таблицы и данные
   - Функции проверки безопасны и не изменяют данные
   - Функции проверки можно запускать многократно

2. **Валидация мигрированных данных**:
   - Функции валидации проверяют структуру данных
   - Функции валидации не изменяют данные (кроме `fix_invalid_adaptations`)
   - Функция `fix_invalid_adaptations` устанавливает статус 'pending' для невалидных адаптаций

3. **Массовая генерация адаптаций**:
   - API endpoint требует аутентификации
   - API endpoint проверяет права доступа (автор или соавтор)
   - API endpoint обновляет статус генерации в метаданных
   - API endpoint обрабатывает ошибки и продолжает генерацию для других уроков
   - API endpoint возвращает статистику генерации

4. **Обработка ошибок**:
   - Ошибки логируются в консоль
   - Ошибки сохраняются в метаданных
   - Ошибки не прерывают выполнение для других уроков
   - Ошибки возвращаются в ответе API

5. **Производительность**:
   - Массовая генерация может занять время для больших курсов
   - Генерация выполняется последовательно для каждого урока и типа
   - Рекомендуется использовать батч-генерацию для небольших курсов
   - Для больших курсов рекомендуется использовать фоновые задачи

6. **Безопасность**:
   - API endpoint проверяет аутентификацию
   - API endpoint проверяет права доступа
   - API endpoint использует RLS политики
   - API endpoint не позволяет изменять чужие адаптации
