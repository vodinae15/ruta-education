# Шаблоны адаптации курсов

Эта папка содержит компоненты для шаблонной системы адаптации курсов под разные типы восприятия.

## Структура

```
/components/adaptation/
├── blocks/                           # Общие блоки (одинаковые для всех типов)
│   ├── BlockWrapper.tsx              # Обертка для всех блоков
│   ├── PracticeBlock.tsx             # Блок 3: Практическое закрепление
│   ├── AttachmentsBlock.tsx          # Блок 4: Углубленное изучение (медиа)
│   └── TestBlock.tsx                 # Блок 5: Итоговое задание (10 вопросов)
│
└── templates/                        # Специфичные шаблоны для типов
    ├── original/                     # Оригинальный тип
    │   ├── FlipCards.tsx             # Блок 1: 6 флип-карточек
    │   └── StructuredText.tsx        # Блок 2: Структурированный текст
    │
    ├── visual/                       # Визуальный тип
    │   ├── VisualDiagram.tsx         # Блок 1: Диаграмма с узлами
    │   └── ComparisonTable.tsx       # Блок 2: Таблица сравнения (4 колонки)
    │
    ├── auditory/                     # Аудиальный тип
    │   ├── AudioUploadBlock.tsx      # Блок 1: Аудио-обзор темы
    │   └── AudioCards.tsx            # Блок 2: 6 аудио-карточек
    │
    └── kinesthetic/                  # Кинестетический тип
        ├── GoalsChecklist.tsx        # Блок 1: Цели + интерактивный чек-лист
        └── PracticalText.tsx         # Блок 2: Практико-ориентированный текст
```

## 4 типа адаптации

### 1. Оригинальный (Original)
- **Блок 1**: 6 интерактивных флип-карточек (термин → объяснение)
- **Блок 2**: Структурированный текст с нумерованными разделами
- **Блоки 3-5**: Стандартные (практика, вложения, тест)

### 2. Визуальный (Visual)
- **Блок 1**: Визуальная диаграмма с 5-8 узлами и связями между ними
- **Блок 2**: Таблица сравнения (4 колонки: Понятие, Определение, Ключевые признаки, Пример)
- **Блоки 3-5**: Стандартные

### 3. Аудиальный (Auditory)
- **Блок 1**: Аудио-блок для загрузки обзора темы (3-5 минут)
  - Автор может заменить на текст если не загрузил аудио
- **Блок 2**: 6 аудио-карточек (термин + аудио-объяснение)
- **Блоки 3-5**: Стандартные

### 4. Кинестетический (Kinesthetic)
- **Блок 1**: Структура целей с интерактивным чек-листом
  - Отслеживание прогресса
  - Иерархические подцели
- **Блок 2**: Практико-ориентированный текст с акцентом на действия
- **Блоки 3-5**: Стандартные

## Стандартные блоки (3-5)

Эти блоки одинаковые для всех 4 типов адаптации:

- **Блок 3**: Практическое закрепление - практические задания
- **Блок 4**: Углубленное изучение - все медиа-вложения из оригинального урока (видео, аудио, документы, изображения)
- **Блок 5**: Итоговое задание - тест из 10 вопросов (одна попытка)

## Использование

### Пустые шаблоны (скелетоны)

Все компоненты поддерживают параметр `isEmpty={true}`, который показывает пустую структуру шаблона:

```tsx
import { FlipCards } from "@/components/adaptation/templates/original/FlipCards"

// Пустой шаблон (для демонстрации автору)
<FlipCards isEmpty={true} />

// Заполненный контентом
<FlipCards isEmpty={false} cards={[...]} />
```

### Страница демонстрации

Посмотреть все шаблоны можно на странице:
```
/app/adaptation-preview/page.tsx
```

## Возможности редактирования для автора

После генерации AI автор может:

✅ **Может**:
- Редактировать любой текст (вступления, заголовки, контент)
- Редактировать содержимое карточек, таблиц, чек-листов
- Перемещать вложения между блоками (drag & drop)
- Заменить аудио-блок на текстовый (для аудиального типа)

❌ **Не может**:
- Изменять тип адаптации
- Добавлять/удалять карточки (фиксированное количество)
- Удалять обязательные блоки (1-5)
- Менять структуру шаблона

## Технические детали

### Флип-карточки (3D анимация)

CSS для флип-карточек добавлен в `/app/globals.css`:

```css
.perspective-1000 {
  perspective: 1000px;
}

.transform-style-3d {
  transform-style: preserve-3d;
}

.backface-hidden {
  backface-visibility: hidden;
}

.rotate-y-180 {
  transform: rotateY(180deg);
}
```

### Адаптивность

Все шаблоны адаптивны:
- **Мобайл**: одна колонка, карточный формат
- **Планшет (sm)**: 2 колонки
- **Десктоп (lg)**: 3-4 колонки (в зависимости от компонента)

### Типизация

Все компоненты полностью типизированы с TypeScript:

```typescript
interface FlipCardData {
  id: string
  front: string
  back: string
}

interface FlipCardsProps {
  isEmpty?: boolean
  cards?: FlipCardData[]
}
```
