# 🔧 Исправление ошибки "JWT expired"

## Проблема
При работе с API адаптации возникала ошибка "JWT expired" (истекший токен аутентификации), которая не обрабатывалась корректно.

## Решение
Добавлена комплексная обработка ошибки истекшего JWT токена во всех местах, где происходят запросы к базе данных.

## Изменения

### 1. Добавлены вспомогательные функции

**`isJwtExpiredError(error)`** - проверяет, является ли ошибка ошибкой истекшего JWT:
- Проверяет код ошибки `PGRST301`
- Проверяет сообщение на наличие "jwt expired", "expired jwt", "token expired"
- Проверяет поля `details` и `hint`

**`handleDatabaseError(error, context)`** - обрабатывает ошибку БД с проверкой JWT:
- Определяет, является ли ошибка ошибкой истекшего JWT
- Возвращает понятное сообщение для пользователя
- Логирует контекст ошибки

### 2. Добавлена обработка во всех критических местах

#### ✅ Аутентификация
- `supabase.auth.getUser()` - проверка авторизации пользователя

#### ✅ Запросы к базе данных
- `course_lessons` - получение урока
- `courses` - получение курса
- `course_collaborators` - проверка прав соавтора
- `analyze_lesson_materials` (RPC) - анализ материалов урока
- `lesson_adaptations` - проверка существующей адаптации
- `lesson_adaptations` - удаление старой адаптации (forceRegenerate)
- `lesson_adaptations` - создание новой адаптации
- `lesson_adaptation_metadata` - обновление метаданных

### 3. Формат ответа при ошибке JWT

При обнаружении ошибки истекшего JWT возвращается:
```json
{
  "success": false,
  "error": "Сессия истекла. Пожалуйста, перезайдите в систему.",
  "code": "JWT_EXPIRED"
}
```
HTTP статус: `401 Unauthorized`

## Результат

✅ Все критические операции с БД теперь проверяют ошибку истекшего JWT  
✅ Пользователь получает понятное сообщение об ошибке  
✅ Система корректно обрабатывает истекшие сессии  
✅ Логирование помогает отследить, где именно произошла ошибка  

## Примечания

- Ошибки при обновлении метаданных (некритичные операции) логируются, но не прерывают процесс
- Ошибка JWT expired может возникнуть, если пользователь долго не был активен
- Рекомендуется обновлять токен через middleware перед каждым запросом
