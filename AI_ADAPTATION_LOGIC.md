# 🤖 Логика адаптации курса с ИИ

## 📋 Обзор

Система адаптации контента использует ИИ для персонализации уроков под тип обучения студента. Адаптация происходит в реальном времени при открытии урока студентом.

## 🔄 Процесс адаптации

### 1. **Триггер адаптации**

Адаптация запускается автоматически при:
- Открытии страницы урока (`/course/[courseId]/learn`)
- Выборе урока из списка
- Загрузке данных урока и сессии студента

**Файл**: `app/course/[courseId]/learn/page.tsx`

```typescript
useEffect(() => {
  if (lessons.length > 0 && studentSession?.student_type && currentLessonData) {
    adaptContent(currentLessonData)
  }
}, [lessons, studentSession, currentLesson])
```

### 2. **Подготовка данных для адаптации**

Перед отправкой в ИИ данные урока формируются в структурированный формат:

```typescript
const lessonContent = {
  title: lessonData.title,
  description: lessonData.description,
  blocks: lessonData.blocks || [], // Массив блоков урока
  materials: [],
  tests: []
}
```

**Структура блока урока**:
```typescript
{
  title: string,      // Название блока
  content: string,    // Содержимое блока
  type: string        // Тип блока (introduction, main_block_1, etc.)
}
```

### 3. **API запрос к ИИ**

**Endpoint**: `/api/ai-adaptation`  
**Метод**: POST  
**Файл**: `app/api/ai-adaptation/route.ts`

#### Параметры запроса:
```typescript
{
  lessonContent: {
    title: string,
    description?: string,
    blocks: Array<{ title: string; content: string; type: string }>,
    materials?: string[],
    tests?: string[]
  },
  studentType: 'visual' | 'auditory' | 'kinesthetic' | 'visual-analytical' | 'auditory-empathetic' | 'kinesthetic-practical',
  lessonId: string
}
```

#### Ответ API:
```typescript
{
  success: true,
  adaptedContent: {
    block1: { ... },
    block2: { ... },
    block3: { ... },
    block4: { ... },
    block5: { ... }
  },
  lessonId: string,
  studentType: string
}
```

## 🤖 Интеграция с ИИ (OpenRouter)

### Конфигурация

**Сервис**: OpenRouter API  
**Модель**: Claude 3.5 Sonnet  
**API URL**: `https://openrouter.ai/api/v1/chat/completions`

**Параметры запроса**:
```typescript
{
  model: 'anthropic/claude-3.5-sonnet',
  messages: [
    {
      role: 'system',
      content: 'Ты - эксперт по адаптации образовательного контента. Всегда отвечай только в формате JSON без дополнительных комментариев.'
    },
    {
      role: 'user',
      content: prompt // Промпт для адаптации
    }
  ],
  temperature: 0.7,
  max_tokens: 2000
}
```

### Генерация промпта

Промпт генерируется функцией `getAdaptationPrompt()` в зависимости от типа студента:

1. **Базовый промпт** - общая информация об уроке:
   - Название урока
   - Описание
   - Блоки урока (содержимое)
   - Дополнительные материалы

2. **Специфичный промпт** - инструкции по адаптации для конкретного типа:
   - Структура из 5 блоков
   - Особенности форматирования
   - Требования к контенту
   - Формат JSON ответа

### Обработка ответа ИИ

1. **Парсинг JSON**:
   ```typescript
   const adaptedContent = JSON.parse(aiResponse)
   ```

2. **Валидация структуры**:
   - Проверка наличия всех 5 блоков
   - Проверка корректности формата данных
   - Обработка ошибок парсинга

3. **Возврат результата**:
   ```typescript
   return NextResponse.json({
     success: true,
     adaptedContent,
     lessonId,
     studentType
   })
   ```

## 🎨 Рендеринг адаптированного контента

**Функционал**:
- Отображение адаптированного контента
- Отслеживание прогресса студента
- Сохранение прогресса
- Отслеживание просмотров блоков
- Интерактивные элементы

```typescript

  courseId={params.courseId}
  lessonId={currentLessonData.id}
  lessonBlocks={currentLessonData.blocks || []}
  onProgressUpdate={(progress, completedBlocks) => {...}}
  onSaveProgress={async (progressData) => {...}}
/>
```

### Рендеринг блоков

Каждый блок рендерится в зависимости от типа студента:

1. **Block 1** - Введение/Содержание
   - Визуал: Структурированный список
   - Аудиал: Атмосферная подводка
   - Кинестетик: Интерактивное содержание

2. **Block 2** - Основная теория
   - Визуал: Разбивка на подблоки с заголовками
   - Аудиал: Аудио-адаптированный контент с историями
   - Кинестетик: Теория с интерактивными элементами

3. **Block 3** - Соотнесение понятий
   - Визуал: Пары понятие-определение
   - Аудиал: Пары с голосовыми подсказками
   - Кинестетик: Перетаскиваемые элементы

4. **Block 4** - Продолжение теории
   - Визуал: Дополнительные подблоки
   - Аудиал: Аудио-материалы и диалоги
   - Кинестетик: Интерактивные задания

5. **Block 5** - Заключение
   - Визуал: Схема урока
   - Аудиал: Голосовое резюме
   - Кинестетик: Чек-лист и памятка

### Интерактивные элементы

#### Для визуалов:
- Интерактивные диаграммы (`InteractiveDiagram`)
- Структурированные списки
- Схемы и таблицы
- Визуальные акценты

#### Для аудиалов:
- Эмоциональный текст
- История и диалоги
- Голосовые подсказки
- Атмосферные элементы

#### Для кинестетиков:
- Интерактивные кнопки
- Перетаскиваемые элементы
- Чек-листы
- Практические задания

## 📊 Отслеживание прогресса

### Автоматическое отслеживание

1. **Отслеживание просмотров блоков**:
   - При открытии блока отправляется запрос к `/api/track-block-view`
   - Записывается время начала просмотра
   - При закрытии обновляется время просмотра

2. **Сохранение прогресса**:
   - Автоматическое сохранение каждые 30 секунд
   - Сохранение при завершении блока
   - Сохранение при выходе из урока

### API для отслеживания

**Endpoint**: `/api/track-block-view`  
**Метод**: POST

```typescript
{
  courseId: string,
  lessonId: string,
  blockId: string,
  blockType: string,
  timeSpent: number
}
```

**Endpoint**: `/api/save-progress`  
**Метод**: POST

```typescript
{
  courseId: string,
  lessonId: string,
  progressData: {
    completedBlocks: string[],
    testResults: { [key: string]: string },
    lastUpdated: string
  },
  timeSpent: number
}
```

## 🔍 Обработка ошибок

### Типы ошибок

1. **Ошибка загрузки урока**:
   - Отображение оригинального контента
   - Логирование ошибки

2. **Ошибка адаптации**:
   - Fallback к оригинальному контенту
   - Сообщение об ошибке студенту
   - Логирование ошибки

3. **Ошибка парсинга JSON**:
   - Попытка исправить формат
   - Fallback к оригинальному контенту
   - Логирование ошибки

### Логирование

Все ошибки логируются в консоль:
```typescript
console.error('Adaptation API error:', error)
console.error('JSON parse error:', parseError)
console.error('Error adapting content:', error)
```

## 🚀 Оптимизация

### Кэширование

- Адаптированный контент сохраняется в состоянии компонента
- Повторная адаптация не выполняется для одного урока
- Кэширование на уровне сессии студента

### Ленивая загрузка

- Адаптация выполняется только при необходимости
- Блоки загружаются по требованию
- Оптимизация запросов к ИИ

### Производительность

- Асинхронная обработка
- Не блокирует рендеринг страницы
- Показ индикатора загрузки во время адаптации
