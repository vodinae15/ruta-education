# 🔧 ИСПРАВЛЕНИЕ: Загрузка уроков на странице адаптации

## Проблема
На странице адаптации курса отображается "Уроков найдено: 0", хотя уроки в курсе есть.

## Причина
Уроки могут храниться в разных местах:
1. **В поле `modules.lessons`** таблицы `courses` (основной источник для конструктора курсов)
2. **В поле `course_data.lessons`** таблицы `courses` (старый формат)
3. **В таблице `course_lessons`** (отдельная таблица для уроков)

Функция `loadLessonsFromDB` проверяла только таблицу `course_lessons`, но не проверяла поля `modules.lessons` и `course_data.lessons`.

## Решение
Обновлена функция `loadLessonsFromDB` в `app/course/[courseId]/adaptation/page.tsx`:

1. **Добавлена проверка поля `modules.lessons`** (первый приоритет)
2. **Добавлена проверка поля `course_data.lessons`** (второй приоритет)
3. **Добавлена проверка таблицы `course_lessons`** (третий приоритет)
4. **Добавлено поле `modules` в запрос SELECT** для загрузки данных курса
5. **Добавлено подробное логирование** для диагностики проблем
6. **Улучшена обработка разных форматов уроков** (из конструктора и из таблицы)

## Изменения в коде

### 1. Обновлен запрос к БД
```typescript
const { data: courseData, error: courseError } = await supabase
  .from("courses")
  .select("id, title, description, status, author_id, course_data, modules") // Добавлено modules
  .eq("id", courseId)
  .single()
```

### 2. Обновлена функция `loadLessonsFromDB`
- Проверяет `courseData.modules.lessons` первым
- Проверяет `courseData.course_data.lessons` вторым
- Проверяет таблицу `course_lessons` третьим
- Добавлено подробное логирование для диагностики

### 3. Улучшена обработка форматов уроков
- Правильно обрабатывает уроки из конструктора (структура `CourseLesson`)
- Правильно обрабатывает уроки из таблицы `course_lessons`
- Корректно преобразует поле `order` в `order_index`

   - Есть ли поле `modules.lessons` в данных курса
   - Есть ли записи в таблице `course_lessons` для этого курса

## Возможные проблемы

### 1. Уроки не найдены в `modules.lessons`
**Причина**: Курс был создан до добавления поля `modules`, или уроки хранятся в другом формате.

**Решение**: Проверьте структуру данных курса в Supabase Dashboard:
- Откройте таблицу `courses`
- Найдите курс по ID
- Проверьте поле `modules` и его структуру
- Проверьте поле `course_data` и его структуру

### 2. Уроки не найдены в таблице `course_lessons`
**Причина**: Уроки не были мигрированы из `modules.lessons` в таблицу `course_lessons`.

**Решение**: Уроки должны загружаться из `modules.lessons`, это нормально. Если уроки не загружаются, проверьте:
- Правильность структуры данных в поле `modules`
- Есть ли массив `modules.lessons` в данных курса
- Правильность преобразования формата уроков

### 3. Проблемы с RLS (Row Level Security)
**Причина**: Политики RLS могут блокировать доступ к данным.

**Решение**: Проверьте политики RLS для таблицы `courses`:
- Автор должен иметь доступ к своим курсам
- Соавтор должен иметь доступ к курсам, где он является соавтором
