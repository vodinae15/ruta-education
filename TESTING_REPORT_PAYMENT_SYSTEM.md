## ⚠️ НАЙДЕННЫЕ ПРОБЛЕМЫ И НЕДОРАБОТКИ

### 1. КРИТИЧЕСКИЕ ПРОБЛЕМЫ

#### ❌ Проблема #1: Отсутствует возможность редактировать `has_feedback` в PricingEditor
**Файл**: `components/ui/pricing-editor.tsx`
**Описание**: В компоненте нет поля для редактирования флага `has_feedback`, хотя это важная функция тарифа.
**Влияние**: Автор не может изменить, есть ли обратная связь в тарифе после создания.
**Решение**: Добавить checkbox для `has_feedback` в компонент (кроме бесплатного тарифа).

#### ❌ Проблема #2: В `stream-launch.ts` обновляются ВСЕ покупки, даже с уже открытым доступом
**Файл**: `lib/stream-launch.ts` (строки 61-90)
**Описание**: При запуске потока обновляются все покупки со статусом `completed`, даже если `access_granted_at` уже установлен.
**Влияние**: Может перезаписать дату открытия доступа для студентов, у которых доступ уже был открыт ранее.
**Решение**: Фильтровать покупки, у которых `access_granted_at` меньше даты старта потока или NULL.

#### ❌ Проблема #3: В `course-access-check.ts` не проверяется `access_granted_at` для покупок
**Файл**: `lib/course-access-check.ts` (строки 92-147)
**Описание**: При проверке доступа для потоковых курсов не учитывается поле `access_granted_at` из покупок. Проверяется только дата старта курса.
**Влияние**: Студент может получить доступ к потоковому курсу до того, как автор запустит поток, если дата старта уже прошла, но автор еще не нажал кнопку запуска.
**Решение**: Добавить проверку `access_granted_at` из покупок для потоковых курсов.

### 2. СРЕДНИЕ ПРОБЛЕМЫ

#### ⚠️ Проблема #4: В `pricing/page.tsx` нет обработки ошибок при загрузке тарифов
**Файл**: `app/course/[courseId]/pricing/page.tsx` (строка 78-82)
**Описание**: Если API возвращает ошибку при загрузке тарифов, ошибка не обрабатывается и не показывается пользователю.
**Влияние**: Пользователь не увидит сообщение об ошибке, если тарифы не загрузились.
**Решение**: Добавить обработку ошибок и показ сообщения пользователю.

#### ⚠️ Проблема #5: В `course-constructor/page.tsx` нет валидации даты старта потока
**Файл**: `app/course-constructor/page.tsx` (строки 810-820)
**Описание**: При сохранении режима запуска нет проверки, что дата старта не в прошлом (для потоковых курсов).
**Влияние**: Автор может установить дату старта в прошлом, что приведет к проблемам.
**Решение**: Добавить валидацию даты (должна быть в будущем или сегодня).

#### ⚠️ Проблема #6: В `notifications-list.tsx` нет обработки ошибок при загрузке
**Файл**: `components/ui/notifications-list.tsx` (строки 23-35)
**Описание**: Ошибки при загрузке уведомлений обрабатываются, но нет индикации загрузки для пользователя.
**Влияние**: Пользователь не видит, что уведомления загружаются.
**Решение**: Уже есть состояние `loading`, но можно улучшить UX.

### 3. МЕЛКИЕ НЕДОРАБОТКИ

#### 💡 Улучшение #1: В `pricing-card.tsx` можно добавить визуальное выделение текущего тарифа
**Файл**: `components/ui/pricing-card.tsx`
**Описание**: Нет визуального выделения тарифа, который уже куплен и активен.
**Влияние**: Пользователь может не понять, какой тариф у него активен.
**Решение**: Добавить prop `isCurrentTariff` и визуальное выделение.

#### 💡 Улучшение #2: В `dashboard/page.tsx` можно добавить информацию о количестве студентов, ожидающих запуска потока
**Файл**: `app/dashboard/page.tsx`
**Описание**: При отображении потокового курса можно показать, сколько студентов уже купили тариф и ждут запуска.
**Влияние**: Автор не видит, сколько студентов готовы к запуску.
**Решение**: Добавить запрос количества покупок для потоковых курсов.

#### 💡 Улучшение #3: В `pricing/page.tsx` можно добавить информацию о том, какой тариф уже куплен
**Файл**: `app/course/[courseId]/pricing/page.tsx`
**Описание**: Нет явного указания, какой тариф активен (если куплено несколько).
**Влияние**: Пользователь может не понять, какой тариф у него активен.
**Решение**: Добавить визуальное выделение активного тарифа.
