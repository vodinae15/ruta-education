# 📚 Структура проекта Ruta Education

## 🎯 Обзор проекта

**Ruta Education** - это образовательная платформа с двойной персонализацией:
- **8 типов авторов** с персонализированными конструкторами курсов
- **9 типов студентов** с адаптивным контентом
- **72 подхода** к обучению и прохождению курсов

## 🏗️ Архитектура проекта

### Технологический стек
- **Framework**: Next.js 14 (App Router)
- **Language**: TypeScript
- **Database**: Supabase (PostgreSQL)
- **Authentication**: Supabase Auth
- **Styling**: Tailwind CSS 4.9
- **UI Components**: Radix UI + shadcn/ui
- **State Management**: React Hooks + Context API
- **API**: Next.js API Routes

## 🗄️ База данных

### Основные таблицы

#### 1. `author_profiles`
Профили авторов курсов с результатами теста
- `id` (UUID)
- `user_id` (UUID) - связь с auth.users
- `author_type` (TEXT) - тип автора (8 типов)
- `communication_style` (TEXT) - стиль коммуникации
- `motivation` (TEXT) - мотивация
- `barrier` (TEXT) - барьер
- `test_results` (JSONB) - результаты теста

#### 2. `courses`
Курсы
- `id` (UUID)
- `author_id` (UUID) - автор курса
- `title` (TEXT)
- `description` (TEXT)
- `status` (TEXT) - draft/published/archived
- `is_published` (BOOLEAN)
- `launch_mode` (TEXT) - stream/permanent
- `stream_start_date` (TIMESTAMP)
- `author_type` (TEXT) - тип автора курса
- `personalization_settings` (JSONB)

#### 3. `course_lessons`
Уроки курсов
- `id` (UUID)
- `course_id` (UUID)
- `title` (TEXT)
- `description` (TEXT)
- `order_index` (INTEGER)
- `blocks` (JSONB) - блоки урока
- `completed` (BOOLEAN)

#### 4. `students`
Студенты
- `id` (UUID)
- `user_id` (UUID) - связь с auth.users
- `email` (TEXT) - уникальный email
- `student_type` (TEXT) - тип студента (9 типов)
- `test_results` (JSONB) - результаты теста

#### 5. `student_course_access`
Доступ студентов к курсам
- `id` (UUID)
- `student_id` (UUID)
- `course_id` (UUID)
- `first_accessed_at` (TIMESTAMP)
- `last_accessed_at` (TIMESTAMP)
- `progress` (JSONB) - прогресс прохождения

#### 6. `course_pricing`
Тарифы курсов
- `id` (UUID)
- `course_id` (UUID)
- `title` (TEXT)
- `price` (DECIMAL)
- `currency` (TEXT)
- `has_feedback` (BOOLEAN) - обратная связь от автора
- `order_index` (INTEGER)

#### 7. `course_purchases`
Покупки курсов
- `id` (UUID)
- `course_id` (UUID)
- `student_id` (UUID)
- `pricing_id` (UUID)
- `purchase_status` (TEXT) - pending/completed/cancelled
- `purchased_at` (TIMESTAMP)

#### 8. `course_collaborators`
Соавторы курсов
- `id` (UUID)
- `course_id` (UUID)
- `collaborator_user_id` (UUID)
- `role` (TEXT) - collaborator
- `created_at` (TIMESTAMP)

#### 9. `student_sessions`
Сессии студентов (прогресс)
- `id` (UUID)
- `course_id` (UUID)
- `student_identifier` (TEXT)
- `student_type` (TEXT)
- `progress` (JSONB)
- `progress_percentage` (INTEGER)
- `current_module` (INTEGER)

#### 10. `student_test_results`
Результаты тестов студентов
- `id` (UUID)
- `student_id` (UUID)
- `course_id` (UUID)
- `test_id` (UUID)
- `result` (JSONB)
- `created_at` (TIMESTAMP)

## 🔐 Система аутентификации

### Провайдер аутентификации
- `lib/auth.tsx` - AuthProvider с использованием Supabase Auth
- Проверка сессии через `supabase.auth.getSession()`
- Слушатель изменений через `onAuthStateChange()`

### Middleware
- `middleware.ts` - обновление сессии на каждом запросе
- Использует `lib/supabase/middleware.ts` для работы с cookies

### Типы пользователей
- **Авторы (Teachers)**: `user_metadata.user_type === "teacher"`
- **Студенты (Students)**: по умолчанию, если не teacher

## 📝 Система типов

### Типы авторов (8 типов)

1. **Оратор** - Стиль: "говорю"
   - Приоритет: аудио/видео формат
   - Подсказки: голосовой ввод, минимальные требования к тексту

2. **Методист** - Стиль: "структурирую"
   - Приоритет: текст, схемы, чек-листы
   - Подсказки: подробные шаблоны, расширенные блоки

3. **Автор на вдохновении** - Стиль: "миксую"
   - Приоритет: видео, текст, изображения
   - Подсказки: гибкие шаблоны, творческий подход

4. **Первопроходец** - Стиль: "миксую" + инновации
   - Приоритет: видео, текст, файлы
   - Подсказки: экспериментальные форматы

5. **Загруженный эксперт** - Стиль: "говорю" + ограниченное время
   - Приоритет: аудио, текст, файлы
   - Подсказки: быстрая сборка, голосовой ввод

6. **Педагог с эмпатией** - Стиль: "говорю" + эмпатия
   - Приоритет: видео, текст, аудио
   - Подсказки: подсказки по фидбэку, блоки вопросов

7. **Практик-рационал** - Стиль: "пишу" + практика
   - Приоритет: текст, задачи, файлы
   - Подсказки: шаблон проблема-решение, четкие задания

8. **Интуитивный автор** - Стиль: "миксую" + интуиция
   - Приоритет: изображения, видео, текст
   - Подсказки: свободная сборка, визуальные форматы

### Типы студентов (9 типов)

1. **Визуал** - Визуальное восприятие
2. **Аудиал** - Аудиальное восприятие
3. **Кинестетик** - Тактильное восприятие
4. **Визуал-Аналитик** - Визуал + аналитическая обратная связь
5. **Аудиал-Эмпат** - Аудиал + эмпатическая обратная связь
6. **Кинестетик-Практик** - Кинестетик + практическая обратная связь
7. **Визуал-Эмпат** - Визуал + эмпатическая обратная связь
8. **Аудиал-Практик** - Аудиал + практическая обратная связь
9. **Универсальный** - Сбалансированный тип

## 🎨 Конструктор курсов

### Структура урока

Каждый урок состоит из 9 блоков:

1. **Introduction** - Введение
2. **Navigation** - Навигация
3. **Main Block 1** - Основной блок 1
4. **Intermediate Practice** - Промежуточная практика
5. **Main Block 2** - Основной блок 2
6. **Intermediate Test** - Промежуточный тест
7. **Main Block 3** - Основной блок 3
8. **Conclusion** - Заключение
9. **Bonus Support** - Бонус и поддержка (опционально)

### Типы элементов

- `title` - Заголовок
- `text` - Текст
- `video` - Видео
- `audio` - Аудио
- `image` - Изображение
- `task` - Задание
- `test` - Тест
- `file` - Файл

### Персонализация конструктора

- Элементы адаптируются под тип автора
- Подсказки в реальном времени
- Рекомендуемые элементы для каждого блока
- Шаблоны блоков в зависимости от типа автора

## 🤖 Адаптация контента

### Процесс адаптации

1. **Определение типа студента** - через тест
2. **Загрузка контента урока** - из базы данных
3. **Адаптация через AI** - API `/api/ai-adaptation`
4. **Рендеринг адаптированного контента** - компонент `AdaptedContentRenderer`

### API адаптации

- **Endpoint**: `/api/ai-adaptation`
- **Метод**: POST
- **Параметры**:
  - `lessonContent` - контент урока
  - `studentType` - тип студента
  - `lessonId` - ID урока

### Структура адаптированного контента

Адаптированный контент состоит из 5 блоков:
1. **Введение** - адаптированное введение
2. **Основная часть** - адаптированная основная часть
3. **Практика** - адаптированные практические задания
4. **Проверка** - адаптированная проверка знаний
5. **Заключение** - адаптированное заключение

## 💰 Система оплаты

### Тарифы курсов

- Каждый курс может иметь несколько тарифов
- Тарифы имеют разные цены и опции
- Опция обратной связи от автора (`has_feedback`)

### Процесс покупки

1. Студент выбирает тариф
2. Создается запись в `course_purchases`
3. Статус покупки: `pending` → `completed`
4. Предоставляется доступ к курсу

## 👥 Система соавторов

### Управление соавторами

- Автор может добавить соавторов к курсу
- Соавторы имеют права на редактирование
- Компонент: `CourseCollaboratorsManager`

### Проверка доступа

- Функция `canEditCourse()` проверяет доступ
- Проверяет, является ли пользователь автором или соавтором
- Используется во всех операциях редактирования

## 📊 API Endpoints

### Основные endpoints

1. **`/api/course-access`** - Управление доступом к курсам
   - GET - получить список студентов
   - POST - предоставить доступ
   - DELETE - отозвать доступ

2. **`/api/course-pricing`** - Управление тарифами
   - GET - получить тарифы курса
   - PUT - обновить тариф

3. **`/api/course-purchase`** - Покупка курсов
   - POST - создать покупку

4. **`/api/ai-adaptation`** - Адаптация контента
   - POST - адаптировать контент для типа студента

5. **`/api/student-courses`** - Курсы студента
   - GET - получить курсы студента

6. **`/api/stream-launch`** - Запуск потокового курса
   - GET - проверить статус запуска
   - POST - запустить поток

7. **`/api/course-collaborators`** - Управление соавторами
   - GET - получить соавторов
   - POST - добавить соавтора
   - DELETE - удалить соавтора

8. **`/api/save-progress`** - Сохранение прогресса
   - POST - сохранить прогресс студента

## 🔒 Безопасность

### Row Level Security (RLS)

Все таблицы защищены политиками RLS:
- Авторы видят только свои курсы
- Студенты видят только свои данные
- Соавторы видят курсы, к которым имеют доступ

### Проверка доступа

- Функция `canEditCourse()` проверяет доступ перед редактированием
- API endpoints проверяют авторизацию
- Service Role Key используется только на сервере
