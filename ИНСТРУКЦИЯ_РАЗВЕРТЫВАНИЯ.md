# 📋 Пошаговая инструкция развертывания

## 🎯 Что нужно сделать за 3 часа

### ⏰ ЧАС 1: Подготовка проекта (30-40 минут)

#### Шаг 1.1: Загрузите проект в GitHub

1. Откройте [github.com](https://github.com) и войдите в аккаунт
2. Нажмите "+" → "New repository"
3. Название: `ruta-education` (или любое другое)
4. **НЕ** ставьте галочки на README, .gitignore, license
5. Нажмите "Create repository"

6. В терминале на вашем компьютере выполните:

```bash
cd /Users/vbthy/Downloads/ruta-edu-project
git init
git add .
git commit -m "Initial commit"
git branch -M main
git remote add origin https://github.com/ВАШ_ЛОГИН/ruta-education.git
git push -u origin main
```

**Замените `ВАШ_ЛОГИН` на ваш GitHub логин!**

#### Шаг 1.2: Создайте проект Supabase

1. Откройте [supabase.com](https://supabase.com)
2. Нажмите "Start your project" → "Sign up" (или войдите)
3. Нажмите "New Project"
4. Заполните:
   - **Name:** Ruta Education
   - **Database Password:** придумайте надежный пароль (запишите!)
   - **Region:** выберите ближайший
5. Нажмите "Create new project"
6. Подождите 2-3 минуты пока проект создается

7. После создания проекта:
   - Перейдите в **Settings** → **API**
   - Скопируйте и сохраните:
     - **Project URL** (например: `https://xxxxx.supabase.co`)
     - **anon public** ключ
     - **service_role** ключ (внизу страницы, секция "Project API keys")

### ⏰ ЧАС 2: Развертывание на сервере (60-80 минут)

#### Шаг 2.1: Подключитесь к серверу

Откройте терминал и выполните:

```bash
ssh root@93.189.229.85
```

Если спросит пароль, введите пароль от сервера.

#### Шаг 2.2: Установите необходимое ПО

Выполните на сервере (копируйте команды по одной):

```bash
# Обновление системы
apt update && apt upgrade -y

# Установка Node.js
curl -fsSL https://deb.nodesource.com/setup_20.x | bash -
apt install -y nodejs

# Установка PM2
npm install -g pm2

# Установка Nginx
apt install -y nginx

# Установка Git
apt install -y git
```

#### Шаг 2.3: Клонируйте проект

```bash
# Создайте папку для проекта
mkdir -p /var/www/ruta-education
cd /var/www/ruta-education

# Клонируйте проект (замените на ваш GitHub URL!)
git clone https://github.com/ВАШ_ЛОГИН/ruta-education.git .

# Перейдите в папку проекта
cd /var/www/ruta-education
```

#### Шаг 2.4: Установите зависимости

```bash
cd /var/www/ruta-education
npm install
```

Это займет 2-3 минуты.

#### Шаг 2.5: Настройте переменные окружения

```bash
nano .env.production
```

В открывшемся редакторе вставьте (замените на ваши значения из Supabase):

```env
NEXT_PUBLIC_SUPABASE_URL=https://xxxxx.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=ваш_anon_ключ
SUPABASE_SERVICE_ROLE_KEY=ваш_service_role_ключ
NEXT_PUBLIC_DEV_SUPABASE_REDIRECT_URL=https://rutaedve.beget.tech/dashboard
```

**Как сохранить в nano:**
- Нажмите `Ctrl + O` (сохранить)
- Нажмите `Enter` (подтвердить имя файла)
- Нажмите `Ctrl + X` (выйти)

#### Шаг 2.6: Соберите проект

```bash
cd /var/www/ruta-education
npm run build
```

Это займет 2-3 минуты.

#### Шаг 2.7: Запустите приложение

```bash
cd /var/www/ruta-education

# Установите dotenv для загрузки переменных окружения
npm install dotenv --save

# Создайте конфигурацию PM2
cat > ecosystem.config.js << 'EOF'
require('dotenv').config({ path: '.env.production' })

module.exports = {
  apps: [{
    name: 'ruta-education',
    script: 'node_modules/next/dist/bin/next',
    args: 'start',
    cwd: '/var/www/ruta-education',
    instances: 2,
    exec_mode: 'cluster',
    env_file: '.env.production',
    env: {
      NODE_ENV: 'production',
      PORT: 3000
    }
  }]
}
EOF

# Запустите приложение
pm2 start ecosystem.config.js

# Сохраните конфигурацию
pm2 save

# Настройте автозапуск
pm2 startup
```

**Важно:** После команды `pm2 startup` появится команда вида:
```
sudo env PATH=... pm2 startup systemd -u root --hp /root
```

Скопируйте и выполните эту команду!

#### Шаг 2.8: Настройте Nginx

```bash
# Создайте конфигурацию Nginx
nano /etc/nginx/sites-available/ruta-education
```

Вставьте:

```nginx
server {
    listen 80;
    server_name rutaedve.beget.tech;

    location / {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_cache_bypass $http_upgrade;
    }
}
```

Сохраните (`Ctrl+O`, `Enter`, `Ctrl+X`)

```bash
# Активируйте конфигурацию
ln -s /etc/nginx/sites-available/ruta-education /etc/nginx/sites-enabled/

# Удалите дефолтную (если есть)
rm -f /etc/nginx/sites-enabled/default

# Проверьте конфигурацию
nginx -t

# Перезапустите Nginx
systemctl restart nginx
```

#### Шаг 2.9: Установите SSL сертификат

```bash
# Установите Certbot
apt install -y certbot python3-certbot-nginx

# Получите SSL сертификат
certbot --nginx -d rutaedve.beget.tech
```

Следуйте инструкциям:
- Введите email
- Согласитесь с условиями (Y)
- Выберите редирект HTTP на HTTPS (2)

### ⏰ ЧАС 3: Миграция базы данных (40-50 минут)

#### Шаг 3.1: Откройте Supabase Dashboard

1. Откройте [supabase.com](https://supabase.com)
2. Войдите в ваш проект
3. Перейдите в **SQL Editor** (в левом меню)

#### Шаг 3.2: Выполните миграцию

1. В SQL Editor нажмите **"New query"**
2. Откройте файл `scripts/00-complete-migration.sql` на вашем компьютере
3. Скопируйте **весь** содержимое файла
4. Вставьте в SQL Editor
5. Нажмите **"Run"** (или `Ctrl+Enter`)

Подождите 10-20 секунд. Должно появиться сообщение об успехе.

#### Шаг 3.3: Проверьте таблицы

1. В Supabase Dashboard перейдите в **Table Editor**
2. Убедитесь, что созданы таблицы:
   - `author_profiles`
   - `courses`
   - `course_modules`
   - `content_blocks`
   - `students`
   - `student_course_access`
   - `student_sessions`
   - `student_test_results`

#### Шаг 3.4: Проверьте работу приложения

1. Откройте в браузере: `https://rutaedve.beget.tech`
2. Попробуйте зарегистрироваться
3. Попробуйте войти
4. Проверьте основные функции

#### Шаг 3.5: Проверьте логи (если что-то не работает)

На сервере выполните:

```bash
# Просмотр логов приложения
pm2 logs ruta-education

# Или последние 50 строк
pm2 logs ruta-education --lines 50
```

## 🐛 Решение проблем

### Приложение не открывается

1. Проверьте, запущено ли приложение:
   ```bash
   pm2 status
   ```
   Должно быть `online`

2. Проверьте логи:
   ```bash
   pm2 logs ruta-education
   ```

3. Проверьте переменные окружения:
   ```bash
   cat /var/www/ruta-education/.env.production
   ```

### Ошибка подключения к Supabase

1. Проверьте URL и ключи в `.env.production`
2. Убедитесь, что проект Supabase активен
3. Проверьте настройки в Supabase Dashboard → Settings → API

### Ошибка Nginx

```bash
# Проверьте конфигурацию
nginx -t

# Просмотрите логи ошибок
tail -f /var/log/nginx/error.log
```
